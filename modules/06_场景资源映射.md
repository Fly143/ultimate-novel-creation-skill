### 预判执行流程（强制执行）

```
=== 单章写作前置预判流程 ===
1. 读取章节规划表：[书名]/设定/[书名]_章节规划表.md
2. 提取当前章节的预判字段：
   - 基础预判：章节类型、冲突类型、爽点类型
   - 扩展预判（从「核心内容」字段提取）：
     - 新角色出场 → 预加载 `templates/人设卡模板.md`
     - 场景转换 → 预加载 
     - 时间跨度 → 标记时间线更新需求
     - 伏笔埋设 → 标记伏笔追踪需求
     - 场景复杂度判断 → 参考 `templates/章节规划表.md`「场景复杂度说明」
       - 简单：场景数≤1、角色数≤3、无双线/多线叙事
       - 中等：场景数2-3、角色数4-6、适度转折
       - 复杂：场景数≥4、角色数≥7、多线并进
       - 复杂度=复杂时，预加载  + 
3. 对照「章节创作场景映射表」，匹配预判值
4. 加载匹配到的所有参考资源（可能多个场景同时触发）
5. 开始章节创作
```

### 多场景叠加处理

**当一章同时匹配多个场景时，按优先级加载：**

| 优先级 | 场景类型 | 说明 |
|--------|----------|------|
| P0 | 开篇创作 | 黄金三章优先级最高 |
| P1 | 首战场景 | 首爽点决定读者留存 |
| P2 | 爽点设计 | 爽点是爽文核心 |
| P3 | 打斗场景 | 战斗描写需要技巧支撑 |
| P4 | 冲突与反转 | 冲突驱动剧情 |
| P5 | 对话场景 | 对话是常规场景 |
| P6 | 情感描写 | 情感是加分项 |
| P7 | 过渡章节 | 过渡章节技巧要求较低 |

**示例**：第5章「冲突升级 + 打脸爽 + 人际冲突」
- 匹配场景：爽点设计（P2）+ 冲突与反转（P4）
- 加载资源：`references/爽文创作理论.md` +  + 

**多场景加载限制规则**：
- 当匹配场景数量 ≥3 个时，仅加载优先级最高的前2个场景资源
- 目的：避免资源加载过多导致上下文臃肿，影响生成质量

### 质量检查触发机制

| 检查类型 | 触发时机 | 自动加载资源 |
|----------|----------|--------------|
| **生成前检查** | 每章生成前 | `templates/章节质量检查清单.md`（前检查部分） |
| **生成后检查** | 每章生成后 | `templates/章节质量检查清单.md`（后检查部分） |
| **深度检查** | 每5章（第5、10、15、20...章） | `templates/章节质量检查清单.md`（深度检查部分） |
| **创新检查** | 每10章（第10、20、30...章） | `templates/创新深度检查清单.md` |
| **避坑检查** | 发现潜在问题时 |  |

**避坑检查触发条件（明确判断标准）**：
- 连续3章出现同类问题（如：人物性格前后矛盾、实力等级混乱）
- 单章出现2处以上严重问题（如：重大情节漏洞、时间线矛盾）
- 检测到违背世界观设定规则

### 功能依赖链强制检查

**部分功能存在前置依赖，执行前必须先加载用户已产出的设定文件（强制执行）：**

| 功能 | 前置依赖 | 强制加载顺序 |
|------|----------|--------------|
| **单章写作** | 章节规划表、人物设定 | ① `[书名]/设定/[书名]_章节规划表.md` → ② `[书名]/设定/[书名]_人设卡.md` → ③ 开始写作 |
| **爽点设计** | 情节大纲 | ① `[书名]/设定/[书名]_章节规划表.md`（了解爽点分布）→ ② 开始设计 |
| **首战必胜** | 情节大纲、世界观设定 | ① `[书名]/设定/[书名]_章节规划表.md` → ② `[书名]/设定/[书名]_世界观设定.md`（了解力量体系）→ ③ `references/首战必胜逻辑链.md` → ④ 开始创作 |
| **对话场景创作** | 人物设定 | ① `[书名]/设定/[书名]_人设卡.md`（确保角色性格一致）→ ② 开始创作 |
| **打斗场景创作** | 世界观设定、力量体系 | ① `[书名]/设定/[书名]_世界观设定.md`（确保战力体系一致）→ ② `references/打斗场景创作指南.md`（选择战斗模板）→ ③ 开始创作 |
| **节奏诊断** | 已写章节、大纲 | ① `[书名]/正文/第XXX章_*.txt`（已有章节）→ ② `[书名]/设定/[书名]_章节规划表.md` → ③ 开始诊断 |
| **商业可行性评估** | 完整设定和正文 | ① `[书名]/设定/` 下所有设定文件 → ② `[书名]/正文/` 下所有章节 → ③ 开始评估 |
| **修订优化** | 原始设定、待修订内容 | ① `[书名]/设定/[书名]_章节规划表.md` → ② 待修订章节正文 → ③ 开始修订 |
| **连贯性检查** | 设定文件、已写章节 | ① `[书名]/设定/[书名]_人设卡.md` + `[书名]/设定/[书名]_世界观设定.md` → ② `[书名]/正文/` 最近5章 → ③ 开始检查 |
| **冲突升级设计** | 世界观设定、人物设定 | ① `[书名]/设定/[书名]_世界观设定.md`（了解力量体系）→ ② `[书名]/设定/[书名]_人设卡.md`（了解反派设定）→ ③  → ④ 开始设计 |

**执行规则**：
- 检测到用户触发上述功能时，AI 必须按顺序先加载前置的**用户产出文件**
- 若前置文件不存在，必须先引导用户创建对应设定
- `references/` 和 `templates/` 仅在**首次创建设定**时加载，后续创作不重复加载

### 首次创建 vs 后续引用

**区分两类资源加载场景：**

| 场景类型 | 触发条件 | 加载资源类型 | 示例 |
|----------|----------|--------------|------|
| **首次创建设定** | 用户选择创建世界观/人物/大纲等 | `references/` + `templates/` | 创建世界观 → 加载  + `templates/世界观设定文档.md` |
| **后续创作引用** | 用户继续创作、写章节、诊断等 | 用户产出的 `[书名]/设定/` 文件 | 写第5章 → 加载 `[书名]/设定/[书名]_人设卡.md` + `[书名]/设定/[书名]_章节规划表.md` |
| **质量检查** | 生成前/后、每5章、每10章 | `templates/` 检查清单 | 每章生成后 → 加载 `templates/章节质量检查清单.md` |
| **高级技巧应用** | 场景复杂度="复杂" 或 用户主动请求 | `references/` 高级资源 | 复杂场景 → 加载  + `references/高级写作技巧-整合版.md` |

**判断逻辑**：
1. 用户触发某功能时，先判断是否已有对应的用户产出文件
2. 若已有 → 直接加载用户产出文件
3. 若没有 → 先加载 `references/` + `templates/` 引导创建，创建完成后再执行原功能

### 创作阶段切换触发机制

**当创作进度进入新阶段时，自动加载对应资源包：**

| 阶段切换 | 触发条件 | 自动加载资源包 |
|----------|----------|----------------|
| **进入立项阶段** | 用户选择"新开小说" | `references/书名生成策略.md`、`templates/项目初始化模板.md` |
| **进入世界观构建** | 立项完成，开始设定世界 | `templates/世界观设定文档.md` |
| **进入人物设计** | 世界观完成，开始设计人物 | `templates/人设卡模板.md` |
| **进入大纲生成** | 人物完成，开始规划大纲 | `references/爽文创作理论.md`、`templates/章节规划表.md` |
| **进入开篇创作** | 大纲完成，开始写第1章 | `references/黄金开篇技巧.md`、`references/首战必胜逻辑链.md`、`templates/黄金开篇模板.md` |
| **进入正文写作** | 开篇完成，进入日常更新 | `templates/单章结构模板.md` + 场景类资源（按章节内容自动匹配） |
| **进入质量优化** | 用户选择"优化"或发现质量问题 | `references/风格把控方法.md`、`templates/章节质量检查清单.md` |
| **进入完结阶段** | 用户选择"完结"或写完最后一章 | `templates/商业可行性评估表.md`、`templates/修订检查清单.md` |

**阶段检测规则**：
- AI 通过对话上下文判断用户当前所处阶段
- 检测到阶段切换时，自动加载对应资源包
- 若用户跳过某阶段（如直接开始写正文），需先加载缺失的前置资源

### 使用说明

**AI 执行流程：**

1. **识别场景**：根据用户需求和当前任务，判断属于哪个场景类型
2. **查找映射表**：在上表中找到对应的触发条件
3. **按需加载**：调用 `read_file` 读取映射表中列出的资源文件
4. **执行任务**：基于加载的资源内容，执行当前任务
5. **质量检查**：任务完成后，根据检查触发机制加载对应检查清单

**注意事项：**
- 不要一次性加载所有资源，只加载当前任务相关的资源
- 依赖链中的前置资源必须优先加载
- 质量检查资源在对应时机自动加载，无需用户主动请求
- 高级资源（大师级能力系统等）仅在复杂场景或用户主动请求时加载
