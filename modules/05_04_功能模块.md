### 4. 情节大纲 

**功能**：生成整体框架、规划章节安排、爽点分布，可选同步到记忆系统
**参考资源**：`` `references/爽文创作理论.md` ``
**模板**：`templates/情节大纲模板.md`、`templates/章节规划表.md`、`templates/百万字规划模板.md`
**记忆系统**：支持自动同步剧情逻辑到 记忆系统/bible/plot_timeline.md

**执行流程（强制按顺序执行）**：

```
=== 情节大纲生成阶段 ===
1. 确定故事结构和阶段划分
2. 规划主线冲突和支线剧情
3. 设计爽点分布节奏
4. 规划伏笔埋设和回收
5. 规划主角成长轨迹（基于世界观设定和人物设定，设计主角从起点到终点的能力/心性/关系成长路径）
6. 保存情节大纲文件：[书名]/设定/[书名]_情节大纲.md

=== 分卷大纲生成（★v9.1 新增） ===
6.1 基于主线大纲的阶段划分，逐卷展开分卷大纲：
    - 每卷50章，标注卷内核心冲突线和预计高潮点
    - 标注本卷出场角色（新角色+旧角色回归）
    - 标注本卷暗线推进节点
6.2 逐卷预登记角色到 [书名]/记忆系统/bible/character_database.md：
    - 填写「按卷预设出场角色」表（角色名+身份+本卷出场范围+预计首次出场章节）
    - 新角色先入「待使用角色（储备池）」，标注预计出场卷号
6.3 保存分卷大纲：[书名]/设定/[书名]_分卷大纲.md

=== 章节规划表生成（强制执行·即章节大纲） ===
7. 基于情节大纲和分卷大纲，生成逐章规划：
   - 每章内容概要
   - 字数目标
   - 章节类型（日常/战斗/情感/过渡）
   - 爽点类型
   - 出场角色
   - 场景
   - 冲突目标
   - 状态标记
8. 保存章节规划表：[书名]/设定/[书名]_章节规划表.md

=== 记忆系统同步（如已初始化） ===
9. 【记忆系统检查】检查记忆系统是否已初始化：
   - 检查路径：[书名]/记忆系统/ 目录是否存在
   - 检查文件：[书名]/记忆系统/bible/plot_timeline.md 是否存在
   - 如果已初始化，执行步骤9.1-9.3；如果未初始化，跳转到步骤10

9.1 【自然语言同步】将大纲内容写入剧情时间线文档：
    - 读取刚生成的情节大纲：[书名]/设定/[书名]_情节大纲.md
    - 读取章节规划表：[书名]/设定/[书名]_章节规划表.md
    - 提取并追加到 [书名]/记忆系统/bible/plot_timeline.md：
      * 主线时间线：将大纲中的主要事件按章节顺序填入（加标签[主线]/[支线X]/[交叉]）
      * 支线状态：创建活跃支线状态表
      * 伏笔清单：同步到 [书名]/记忆系统/bible/chekhovs_gun.md
    - **★ 同步更新角色数据库**：[书名]/记忆系统/bible/character_database.md
      * 根据大纲填写「角色重要性层级」表（主角/核心配角/关键角色/次要/单元/背景）
      * 根据大纲填写「按卷预设出场角色」表（每卷的角色配置，含预计出场章节范围）
      * 将大纲中提到的新角色添加到「待使用角色（储备池）」——后续写前单元案细纲时从此池迁入正式表
      * 仅登记角色名+身份+预计出场卷号，不填详细人设（留给单元案细纲阶段展开）
      * 注意：不需要转换为 JSON；LLM 直接读写 markdown 表格
    - **[书名]/设定/角色储备表.md**：从大纲/章节规划表提取所有已命名角色，同步登记到此文件作为角色池索引

9.2 【更新】添加到进度看板：
    - 标记任务完成：[x] 05_04完成大纲后更新 plot_timeline.md
    - 保存：[书名]/设定/进度看板.md

9.3 【输出】剧情逻辑同步完成报告：
    ```
    ✅ 剧情逻辑已同步到记忆系统

    【已同步】
    ✓ 主线事件已写入 plot_timeline.md
    ✓ 支线已创建独立追踪
    ✓ 伏笔已同步到 chekhovs_gun.md

    【下一步】
    1. 使用05_05模块生成黄金开篇，自动创建首章摘要
    2. 05_06（单章写作）会持续维护剧情时间线和伏笔清单
    ```

10. 完成情节大纲流程
```

**输入信息**：
- 故事主线（核心目标、核心冲突）
- 结构选择（三幕式/英雄之旅/起承转合/多线并行）
- 爽点类型偏好

**输出**：
- 情节大纲：`[书名]/设定/[书名]_情节大纲.md`
- 章节规划表：`[书名]/设定/[书名]_章节规划表.md`
- 剧情逻辑同步报告（如记忆系统已初始化）

**记忆系统集成说明**：

| 功能 | 实现方式 | 触发时机 |
|------|---------|---------|
| **剧情逻辑同步** | 自然语言同步到 plot_timeline.md | 大纲生成完成后自动执行 |
| **上下文关系更新** | 解析章节规划并更新上下文 | 章节规划表生成后自动执行 |
| **伏笔追踪初始化** | 将伏笔规划同步到记忆系统 | 大纲生成后自动执行 |
| **剧情进展追踪** | 每章后自动更新剧情状态 | 每章创作后自动更新（05_06模块） |

**注意事项**：
1. 记忆系统同步是可选的，只有在记忆系统已初始化时才会执行
2. 如果记忆系统未初始化，大纲生成会正常完成，只是不会同步到记忆系统
3. 同步后的剧情逻辑会在每章创作前自动提供当前剧情进展
4. 每章创作后会自动更新剧情状态、伏笔回收情况
5. 如果后续有剧情调整，可以通过05_18模块手动更新剧情逻辑
