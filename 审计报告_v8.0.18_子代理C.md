## 🔴 Critical 发现

### 发现1：05_26 A/G子代理context中bible文档路径缺少{书名}/前缀

- **文件**：`modules/05_26_功能模块.md`
- **行号**：L104-L106（G子代理）, L126-L128（A子代理）
- **严重度**：🔴 Critical
- **P编号**：P40扩展（路径引用完整性）
- **描述**：
  A和G子代理的inline context中，bible文档路径写为 `记忆系统/bible/story_bible.md`（无`{书名}/`前缀），但按SKILL.md L27-29的路径约定，项目文件的bible文档路径应为 `{书名}/记忆系统/bible/story_bible.md`。如果main agent的工作目录未设置在 `{书名}/` 内部，子代理将无法定位到正确的bible文档。

- **现状（G子代理 L103-L106）**：
  ```
  context: 中文回复。项目目录：{工作目录}，章节文件：{书名}/正文/第NNN章_章节名.txt。
    需读取：章节正文、{书名}/设定/{书名}_世界观设定.md、记忆系统/bible/story_bible.md、
            记忆系统/bible/character_arcs.md、记忆系统/bible/chekhovs_gun.md、前1章摘要。
  ```

- **现状（A子代理 L123-L128）**：
  ```
  context: 中文回复。项目目录：{工作目录}，章节文件：{书名}/正文/第NNN章_章节名.txt。
    需读取：章节正文、{书名}/设定/[书名]_世界观设定.md、{书名}/设定/[书名]_人设卡.md、
            {书名}/设定/[书名]_情节大纲.md、记忆系统/bible/story_bible.md、
            记忆系统/bible/character_arcs.md、记忆系统/bible/chekhovs_gun.md。
  ```

- **为什么严重**：其他路径（如 `{书名}/设定/xxx`）都正确带 `{书名}/` 前缀，唯独bible文档路径缺失前缀。这种不一致会让main agent在执行占位符替换时产生歧义——工作目录到底该设为哪个层级？子代理可能因路径错误而静默失败（读空文件），导致审核门禁形同虚设。

- **修复建议**：
  ```diff
  - 记忆系统/bible/story_bible.md、记忆系统/bible/character_arcs.md、记忆系统/bible/chekhovs_gun.md
  + {书名}/记忆系统/bible/story_bible.md、{书名}/记忆系统/bible/character_arcs.md、{书名}/记忆系统/bible/chekhovs_gun.md
  ```
  同样的修复需同时应用于A和G两个子代理的context。

## 🟠 High 发现

### 发现2：05_13占位符格式系统性不一致（使用[]而非{}）

- **文件**：`modules/05_13_功能模块.md`
- **行号**：L19-L34（delegate_task context模板全文）
- **严重度**：🟠 High
- **P编号**：P57（占位符格式跨模块一致性）
- **描述**：
  05_13模块在所有路径中使用 `[书名]` 和 `[工作目录]` 作为占位符（方括号格式），而SKILL.md L266明确约定占位符格式为 `{书名}`、`{工作目录}`（花括号格式）。其他所有子代理源模块（05_12/05_17/05_22/05_25）及05_26均使用花括号格式。

- **现状示例（L19-L34）**：
  ```
  context: |
    项目目录：[工作目录]
    章节文件：[书名]/正文/第XXX章_章节名.txt
    ...
    - [书名]/正文/第XXX章_章节名.txt（章节正文）
    - [书名]/设定/[书名]_人设卡.md（人物设定）
  ```

- **对比（05_12 L24，正确格式）**：
  ```
  context: "中文回复。项目目录：{工作目录}\n\n章节文件：{书名}/正文/第XXX章_章节名.txt...
  ```

- **影响**：Main agent在替换占位符时，如果按 `{...}` 正则模式替换，将无法识别05_13的 `[...]` 格式占位符。子代理F（质量检查）会收到未替换的原始占位符，导致文件路径无效。

- **修复建议**：将05_13模块中 delegate_task context模板内的所有 `[书名]` 替换为 `{书名}`，`[工作目录]` 替换为 `{工作目录}`。注意：文件名模式中的 `[书名]`（如 `[书名]_世界观设定.md`）也应替换为 `{书名}`——文件名中的 `{书名}` 替换后恰好形成实际文件名（如 `我的修仙小说_世界观设定.md`）。
