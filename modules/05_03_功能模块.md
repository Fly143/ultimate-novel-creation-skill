### 3. 人物设计 

**功能**：设计主角人设、构建人物关系网络、规划人物成长弧光，可选同步到记忆系统
**参考资源**：
**模板**：`templates/人设卡模板.md`、`templates/人物关系图谱模板.md`
**记忆系统**：支持自动同步人物档案到 记忆系统/bible/character_arcs.md

**执行流程（强制按顺序执行）**：

```
=== 人物设计阶段 ===
1. 设计主角人设（性格、背景、目标、金手指等）
2. 设计反派和重要配角人设
3. 保存人设卡文件：[书名]/设定/[书名]_人设卡.md

=== 人物关系图谱生成（强制执行） ===
4. 基于已设计的人物，构建人物关系网络：
   - 主角核心关系（导师、宿敌、CP、伙伴等）
   - 反派关系网络
   - 重要配角关系网络
5. 保存人物关系图谱：[书名]/设定/[书名]_人物关系图谱.md

=== 记忆系统同步（如已初始化） ===
6. 【记忆系统检查】检查记忆系统是否已初始化：
   - 检查路径：[书名]/记忆系统/ 目录是否存在
   - 检查文件：[书名]/记忆系统/bible/character_arcs.md 是否存在
   - 如果已初始化，执行步骤6.1-6.2；如果未初始化，跳转到步骤7

6.1 【自然语言同步】将人设内容追加到人物弧线文档：
    - 读取刚生成的人设卡：[书名]/设定/[书名]_人设卡.md
    - 读取人物关系图谱：[书名]/设定/[书名]_人物关系图谱.md
    - 提取每个主要角色的：
      * 核心性格和初始定位
      * 核心动机和弱点
      * 口头禅/标志性特征
      * 与其他角色的初始关系
    - 追加到 [书名]/记忆系统/bible/character_arcs.md：
      * 为每个主要角色创建独立段落的「首次出场」和「初始定位」
      * 写入当前关系状态
    - 注意：不需要转换为 JSON；LLM 直接读写 markdown
    - 注意：05_06（单章写作）会在此后持续更新角色弧线

6.2 【输出】人物同步完成报告：
    ```
    ✅ 人物信息已同步到记忆系统

    【已同步角色】
    ✓ 主角 - 已写入 [书名]/记忆系统/bible/character_arcs.md
    ✓ 主要配角（N人）- 已写入 [书名]/记忆系统/bible/character_arcs.md

    【下一步】
    1. 使用05_04模块生成情节大纲，自动同步到 plot_timeline.md
    2. 05_06（单章写作）会持续维护人物弧线
    ```

7. 完成人物设计流程
```

**输入信息**：
- 角色定位（主角/配角/反派）
- 角色功能（推动剧情/制造冲突/情感支撑等）
- 性格关键词
- 背景设定
- 目标与动机

**输出**：
- 人设卡：`[书名]/设定/[书名]_人设卡.md`
- 人物关系图谱：`[书名]/设定/[书名]_人物关系图谱.md`
- 人物档案同步报告（如记忆系统已初始化）

**记忆系统集成说明**：

| 功能 | 实现方式 | 触发时机 |
|------|---------|---------|
| **人物档案同步** | LLM 写入 character_arcs.md 自然语言 | 人物设计完成后自动执行 |
| **关系图谱同步** | 解析关系图谱并更新记忆系统 | 人物关系图谱生成后自动执行 |
| **人物一致性自检** | LLM 对照 character_arcs.md 语义检查 | 每章创作后自动执行（05_06模块） |

**注意事项**：
1. 记忆系统同步是可选的，只有在记忆系统已初始化时才会执行
2. 如果记忆系统未初始化，人物设计会正常完成，只是不会同步到记忆系统
3. 同步后的人物档案会在每章创作前自动提供人物当前状态
4. 每章创作后会自动检查人物一致性（OOC检查）
5. 如果后续有人物变化，可以通过05_18模块手动更新人物档案
