## 一、总流程（顺序不可跳）

```
步骤1  字数确认（脚本，CJK 统计，不靠 LLM）
```

## 二、步骤2 主代理逻辑自检（★核心新增）

**目的**：在派任何子代理前，主代理利用自己持有的完整上下文（设定全貌+前文+本章写作意图），抓"只有写手视角才看得出"的逻辑洞。
**为什么必须主代理做**：主代理知道设定全貌与伏笔释放节奏，能判断"我是不是把长明和禁点写矛盾了""主角这章该不该知道灯油=阳寿"。子代理（含 G）只拿磁盘文件、且被指示"与设定一致"，会把含糊设定当正确而漏判——所以**设定自打架、信息越级这两类必须由主代理前置查**。

**自检清单（聚焦"主代理独有视角"，逐条过，任一项不通过→主代理直接修正文，不进子代理）**：

1. **设定不自打架**：本章呈现的设定是否与 bible/世界观文档自我一致？（例："灯长明"与"爷爷禁点"须区分=长明是状态，禁点是禁主动激活照见，不能让读者误读为矛盾）
2. **信息越级**：主角是否说了/想了本章尚未该知道的信息？（例：第1章不知灯油=阳寿，却写"抽的是他的寿"→越级，降级为模糊直觉）
3. **前文/蓝图一致性**：本章与已写前文的因果、人物状态是否衔接？是否偏离章节规划表蓝图（出场角色/场景/冲突目标）？
4. **伏笔对齐**：本章是否提前/滞后回收伏笔？新埋伏笔是否与 chekhovs_gun.md 状态一致？

> ⚠️ **与步骤6 子代理 G 的分工（关键，避免重复/互否）**：
> - **本步骤（主代理）查"写手视角"**：设定自打架、信息越级、前文/蓝图衔接、伏笔对齐——这些依赖主代理的完整上下文，子代理拿不到。
> - **步骤6 子代理 G 查"读者盲读视角"**：空间矛盾、时间矛盾、因果断层、视角纯净——G 只拿磁盘文件、以第一次读的读者身份推演常识性 bug。
> - 两者清单**刻意不重叠**：主代理不查空间/时间/因果/视角（那是 G 的活），G 不查设定自打架/信息越级（那是主代理的活）。任一不通过都阻断本章。

**输出**：自检通过则继续；不通过则主代理直接修正文（属剧情逻辑，不需启修复子代理），修完重跑字数+本步骤。

## 三、步骤4-6 子代理审核规格

> 全部通过 `delegate_task` 并行启动（注意 max_concurrent_children=3，超限分批）。
> 每个子代理 `toolsets:["file"]`，context 必含：项目工作目录、章节文件名、所需读取文件清单。
> 子代理**只返回一行**：PASS 或 FAIL: [N]个问题需修复。完整报告写入指定文件，主代理不读报告内容。

### 第一批（步骤4，3 并行）★ G 提前至此批，逻辑洞早暴露早止损
| 子代理 | 职责 | 报告路径 |
|--------|------|----------|
| A 五道门禁 | 引用/一致性/蓝图/未知实体/描写 | [书名]/设定/第NNN章_五道门禁报告.md |
| B 强约束（叙事参谋） | 金手指叙事/时间约束/叙事线监控；**始终PASS**，有风险时附加提示，零风险静默 | [书名]/设定/第NNN章_强约束报告.md |
| G 逻辑因果审查 | ★独立读者视角魔鬼推演，专查空间/时间/因果/视角纯净（读者盲读4项） | [书名]/设定/第NNN章_逻辑因果审查报告.md |

**G 子代理调用规格**：
```
goal: 对第NNN章执行逻辑因果审查（独立读者视角）
context: 中文回复。项目目录：{工作目录}，章节文件：{书名}/正文/第NNN章_章节名.txt。
  需读取：章节正文、设定/世界观设定文档.md、记忆系统/bible/story_bible.md、
          记忆系统/bible/character_arcs.md、记忆系统/bible/chekhovs_gun.md、前1章摘要。
  以"第一次读的读者"身份（只信本章+前文呈现，不预先相信设定全貌）逐条推演，重点查：
  1. 空间矛盾：物体/人物/声音位置是否明确且不矛盾（如灯在主角屋，却写"听见哭声从灯方向来"）
  2. 时间矛盾：跨度/顺序是否合理，有无同时出现在两处
  3. 因果断层：关键情节因→果是否成立，有无机械降神/凭空能力或信息
  4. 视角纯净：有无作者跳视角旁白（主角无此认知的系统/档案/上帝句）
  每条：✅或❌+具体段落引用+矛盾点+修改建议。
  报告写入：{书名}/设定/第NNN章_逻辑因果审查报告.md
  只返回一行：PASS 或 FAIL: [N]个问题需修复
```
> 注意：G 与步骤2 主代理自检**刻意互补、清单不重叠**——G 只查上面4项"读者盲读视角"；设定自打架、信息越级由步骤2 主代理查（子代理拿不到完整设定上下文）。两者都过才算逻辑闭环。
>
> **读取摘要数量差异说明**：G 只读前1章摘要（查本章逻辑自洽只需前文衔接），而子代理 C（读者审稿）读前5章摘要（评估连贯性/吸引力需要更长上下文）。这是故意的设计差异，不是遗漏。

**A 子代理调用规格（五道门禁）**：
```
goal: 对第NNN章执行五道门禁检查（引用/一致性/蓝图/未知实体/描写）

  五道门禁逐条检查：
  1. 引用门禁：对话/叙述中引用的设定、事件、人物是否与已有文档一致
  2. 一致性门禁：人物言行是否与设定性格/当前状态一致（OOC检测）
  3. 蓝图门禁：本章是否偏离大纲规划（出场角色/场景/冲突目标）

  5. 描写门禁：环境/动作/外貌描写是否自洽、是否有明显感官矛盾
  每道门禁：✅或❌+具体段落引用+问题描述+修改建议。
  报告写入：{书名}/设定/第NNN章_五道门禁报告.md
```

## 四、步骤7 结论收集与修复

- 等步骤4/5/6 全部子代理返回（A/B/C/D/E/F/G 共 7 个结论；步骤2 主代理自检已先行 PASS 才到此）。
  - ⚠️ **B（强约束）始终返回 PASS**：B 是叙事参谋（定义于 05_22），不报违规，只在有风险时附加提示，零风险时静默。B 的 PASS/提示不触发修复子代理，不阻断流水线。
- 全 PASS → 标 .done_logic_selfcheck（步骤2主代理自检）/ .done_gates / .done_constraints / .done_logic_causal / .done_review / .done_coherence / .done_rhythm / .done_quality（以及 .done_wordcount/.done_bible/.done_summary 已在步骤1/3 标）→ 标 .done_chapter_NNN。
- 任一 FAIL → 启**独立修复子代理**（delegate_task，toolsets:["file"]）：
  - 读取所有 FAIL 报告 + 章节正文，定位 🔴/❌ 问题，修复正文重写文件。
  - 修复后**重启对应子代理二次审核**（含 G）。二次仍 FAIL → 报告用户，等指示。
  - 主代理不读审核报告内容，修复决策/执行全交独立子代理。

### .done 标记存储规范

**路径规则**：`[书名]/.done/` 目录下，每个标记为一个空文件。

**完整标记文件名清单**（以第 NNN 章为例）：
```

[书名]/.done/第NNN章_constraints.done      ← 步骤4 B（强约束）PASS（始终PASS）
[书名]/.done/第NNN章_logic_causal.done     ← 步骤4 G（逻辑因果审查）PASS
[书名]/.done/第NNN章_review.done           ← 步骤5 C（读者审稿）PASS
[书名]/.done/第NNN章_coherence.done        ← 步骤5 D（连贯性）PASS
```

**规则**：
- 每个子代理/步骤完成后，主代理立即写入对应 `.done` 空文件（文件大小=0）

- `.done` 文件是流水线状态的唯一真相源；重启/恢复时通过检查 `.done` 目录判断某章完成了哪些步骤
