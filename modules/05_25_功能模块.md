### 25. 读者审稿模块（Reader Review）

**功能**：以挑剔的读者视角，对单章进行质量审核。评估故事连贯性、吸引力、爽点密度，检测剧情 bug，输出评分报告。

**触发方式**：
- 手动：`读者审稿` / `审稿` / `审核剧情单元`
- 自动：每章写完后自动触发（在写后流水线 05_26 步骤5 读者审稿阶段，由独立子代理C执行）

---

## 子代理调用规格

> 本模块必须由 `delegate_task` 启动的独立子代理执行。
> 子代理不共享主代理的写作上下文，避免"自己审自己"的偏见。

### 调用方式

```
delegate_task(
  goal: "以挑剔读者视角对单章进行质量审核评分",
  context: "中文回复。项目目录：{工作目录}\n
    章节文件：{书名}/正文/第XXX章_章节名.txt\n
    需要读取的文件：\n
    - 章节正文\n
    - {书名}/设定/{书名}_人设卡.md\n
    - {书名}/设定/{书名}_情节大纲.md\n
    - {书名}/记忆系统/bible/story_bible.md\n
    - {书名}/记忆系统/summaries/（最近5章摘要）\n
    读完文件后，严格按以下流程执行审核：",
  toolsets: ["file"]
)
```

### 需要读取的文件
| 文件 | 用途 |
|------|------|
| 章节正文 | 审核对象 |
| 人设卡 | 检查人物一致性 |
| 情节大纲 | 检查剧情是否偏离 |
| story_bible.md | 检查设定一致性 |
| 前5章摘要 | 检查连贯性 |

### 输出要求
- 将审核报告写入 `{书名}/记忆系统/summaries/ch_NNN_review.md`
- 如有必须修改的问题，按问题类型标记到对应文档（参见步骤3的bug分类映射表）
- **流水线模式（由 05_26 步骤5 调用）**：只返回一行 `PASS` 或 `FAIL: [N]个问题需修复`，完整报告写入文件；判断依据：context 中包含 `mode: pipeline` 标识
- **独立调用模式**（用户手动触发）：返回完整审核报告文本

## 审核流程

### 步骤 1 — 读者审核

按以下 4 个维度对**本章**评分（每项 25 分，共 100 分）：

#### 1. 故事连贯性（25 分）
- 本章是否与前章剧情自然衔接？
- 剧情发展是否符合大纲设定和人物设定？
- 是否存在前后矛盾或逻辑漏洞？
- 结尾是否设置悬念或钩子引出下一章（非必需但加分）？

#### 2. 吸引力与可读性（25 分）
- 本章是否有足够吸引力，让读者想继续看下一章？
- 章节节奏控制是否合理（有张有弛）？
- 本章是否有明确的"章节高潮"或"章节反转"？
- 是否存在拖沓或冗余内容？
- **★风格一致性**：文风是否前后统一、叙事语调是否与类型匹配、对话风格是否区分角色

#### 3. 爽点密度（25 分）
- 本章是否包含爽点（≥1 个）？
- 爽点类型是否匹配本章定位？（开篇章重悬念和格局展示，后续章重碾压和成长）
- 爽点节奏是否与章节结构匹配？

#### 4. 剧情 Bug 检测（25 分）
- 是否存在设定矛盾（如能力上限冲突）？
- 是否存在人物性格崩坏？
- 是否存在情节漏洞（如物品消失、能力突然失效）？
- 是否存在时间线错误或因果关系不成立？

### 步骤 3 — 写入文件
- 将审核报告写入 `{书名}/记忆系统/summaries/ch_NNN_review.md`
- 如有必须修改的问题，按问题类型标记到对应文档：
  - 设定矛盾/能力冲突 → `{书名}/设定/[书名]_世界观设定.md` 或 `story_bible.md`
  - 人物OOC/性格崩坏 → `{书名}/记忆系统/bible/character_arcs.md`
  - 情节漏洞/时间线错误 → `{书名}/记忆系统/bible/plot_timeline.md`
  - 伏笔相关bug → `{书名}/记忆系统/bible/chekhovs_gun.md`
  - 其他问题 → 在报告自身的 `## 待修复问题` 区块记录，由修复子代理读取

## 评分标准

| 等级 | 分数 | 含义 |
|------|------|------|
| 优秀 | 90-100 | 完美！本章可以作为全书的标杆 |
| 良好 | 80-89 | 很棒！但有一些小问题需要调整 |
| 合格 | 70-79 | 还可以，但需要改进核心问题 |
| 不合格 | <70 | 本章有严重问题，建议重写 |

---

## 常见读者吐槽点（经验库）

- "开头就弃了" → 开篇钩子不够强
- "剧情太假了" → 逻辑漏洞或设定冲突
- "主角太蠢了" → 人设崩坏
- "这章水字数" → 内容拖沓、重复
- "没看懂在说什么" → 设定不清晰或逻辑混乱
- "爽点太少了" → 爽点密度不足
- "节奏太慢了" → 节奏拖沓，缺少高潮
