### 15. 商业可行性评估

**功能**：商业价值评估、市场契合度分析、竞品对标

**定位**：完结/阶段性评估工具。全书完稿后或关键卷结束时触发，评估作品的市场潜力。

**触发方式**：
- 手动：`商业评估` / `市场分析`
- 不参与写后流水线（非每章触发）

**参考资源**：`references/爽文五阶段设定框架.md`

---

## 子代理调用规格

> 本模块可通过 `delegate_task` 启动独立子代理执行（推荐，避免主代理偏见），也可由主代理直接执行。

### 调用模板

```
delegate_task(
  goal: "对作品进行商业可行性评估，分析市场契合度、竞品对标、商业潜力，输出评估报告",
  context: "中文回复。项目目录：{工作目录}

    需要读取的文件：
    - {书名}/设定/{书名}_立项定位单.md（题材/情绪标签/目标读者/篇幅规划）
    - {书名}/设定/{书名}_世界观设定.md（世界框架/力量体系）
    - {书名}/设定/{书名}_人设卡.md（主角/配角/反派设定）
    - {书名}/设定/{书名}_情节大纲.md（主线结构/爽点分布）
    - {书名}/正文/（开头5章+最近5章，评估开篇吸引力和当前质量）
    - {书名}/设定/创作状态追踪表.md（进度/数据）
    - {书名}/记忆系统/bible/story_bible.md（项目概况）
    - references/爽文五阶段设定框架.md（评估参考框架）

    按以下流程逐项评估，输出结构化报告。",
  toolsets: ["file"]
)
```

### 输出要求

- 报告保存至：`{书名}/设定/商业可行性评估报告.md`
- 包含6个评估维度的详细分析
- 每个维度给出评分和具体建议
- 提供优化建议并按优先级排序

---

## 执行流程

```
=== 商业可行性评估阶段 ===

1. 确认评估对象
   - 全本评估 / 阶段性评估（卷级别）
   - 评估范围：书名 / 大纲 / 人设 / 开篇 / 已写章节

2. 读取参考框架
   - 读取 `references/爽文五阶段设定框架.md`

3. 【书名吸引力评估】
   - 是否吸引点击：对比同类型热门作品命名模式
   - 是否符合题材调性
   - 是否易记易传播
   - 评分：1-10

4. 【开篇吸引力评估】
   - 前三章是否有钩子（悬念/冲突/反差）
   - 节奏是否紧凑、信息密度是否足够
   - 读者读完前三章后是否有强烈追读意愿
   - 评分：1-10

5. 【市场契合度评估】
   - 是否符合目标平台的读者口味和推荐算法偏好
   - 题材热度判断（上升期/成熟期/衰退期）
   - 与同类作品的差异化程度
   - 评分：1-10

6. 【商业潜力评估】
   - 付费转化潜力：爽点密度/章节钩子/情绪价值
   - 连载可持续性：世界观扩展性/支线储备/篇幅规划
   - IP改编潜力：视觉化程度/角色辨识度/世界观独特性
   - 评分：1-10

7. 【竞品对标分析】
   - 同类型Top3作品对比
   - 本作优势 vs 短板
   - 差异化策略建议

8. 生成评估报告 → 保存至 {书名}/设定/商业可行性评估报告.md
9. 提供优化建议，按优先级排序（高/中/低）
```

## 常见评估参考

| 评估维度 | 良好信号 | 危险信号 |
|----------|----------|----------|
| 书名 | 3-7字、含题材关键词、有记忆点 | 过长/过短、无辨识度、难搜索 |
| 开篇 | 300字内出冲突、第1章有钩子 | 大段背景介绍、无冲突开端 |
| 市场 | 题材处于上升期、有明确读者画像 | 题材已饱和、无差异化 |
| 商业 | 每5章有爽点、章末有钩子 | 爽点间隔>3章、无追读动机 |

---

## 注意事项

1. **评估时机**：建议全书完稿后做完整评估，也可每卷结束时做阶段性评估
2. **独立执行**：推荐通过 delegate_task 独立子代理执行，避免写作语境影响商业判断
3. **客观性**：评估应基于市场数据和同类对比，而非主观喜好
4. **报告归档**：报告保存至设定/目录，供完结后回顾
