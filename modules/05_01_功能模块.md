### 1. 项目初始化 

**功能**：生成书名、创建立项定位单、推荐创作模式、可选初始化记忆系统
**参考资源**：`references/书名生成策略.md`、`references/爽文五阶段设定框架.md`
**模板**：
**记忆系统**：自动初始化 `记忆系统/` 自然语言记忆系统（默认启用，实现长篇连载连贯性管理）

**执行流程（强制按顺序执行）**：
```
=== 项目初始化阶段 ===
1. 读取 `references/书名生成策略.md`，根据用户输入生成书名候选
2. 读取 ，帮助用户确定题材类型
3. 读取 ，了解五阶段创作框架
4. 用户确认书名后，创建项目目录结构：[书名]/
5. 生成立项定位单，包含：题材、情绪标签、目标读者、篇幅规划、每章字数（默认2000字，用户可自定义）、创作模式推荐
6. 保存立项定位单：[书名]/设定/[书名]_立项定位单.md
7. 初始化进度看板：[书名]/设定/进度看板.md

=== 记忆系统初始化（默认启用） ===
8. 【自动初始化】记忆系统（默认启用，无需询问用户）：
   记忆系统自动管理人物一致性、剧情追踪、伏笔管理，是长篇连载的基础设施。
   直接执行步骤 8.1-8.4，无需用户确认。

8.1 【创建】记忆系统目录结构：
    - 创建 [书名]/记忆系统/bible/
    - 创建 [书名]/记忆系统/summaries/
    - 创建 [书名]/记忆系统/phases/
    - 创建 [书名]/记忆系统/volumes/
    - 创建 [书名]/记忆系统/constraints/

8.2 【复制模板】从 skill 的 `memory-system/` 目录复制模板到项目目录：
    ⚠️ 【重要】skill 内的 `memory-system/` 模板文件是只读的，永远不要直接修改！
       所有读写操作都针对 `[书名]/记忆系统/`（项目目录下的副本）。

    a) 复制 bible 模板（从 skill → 项目）：
       - 读取 skill 的 `memory-system/bible/story_bible.md`，写入 `[书名]/记忆系统/bible/story_bible.md`
       - 读取 skill 的 `memory-system/bible/character_arcs.md`，写入 `[书名]/记忆系统/bible/character_arcs.md`
       - 读取 skill 的 `memory-system/bible/plot_timeline.md`，写入 `[书名]/记忆系统/bible/plot_timeline.md`
       - 读取 skill 的 `memory-system/bible/chekhovs_gun.md`，写入 `[书名]/记忆系统/bible/chekhovs_gun.md`

    b) 复制其他模板（从 skill → 项目）：
       - 读取 skill 的 `memory-system/summaries/_template.md`，写入 `[书名]/记忆系统/summaries/_template.md`
       - 读取 skill 的 `memory-system/phases/_template.md`，写入 `[书名]/记忆系统/phases/_template.md`
       - 读取 skill 的 `memory-system/volumes/_template.md`，写入 `[书名]/记忆系统/volumes/_template.md`

    c) 复制约束模板（从 skill → 项目）：
       - 读取 skill 的 `templates/constraints/cheat_constraints_template.md`，写入 `[书名]/记忆系统/constraints/cheat_constraints.md`
       - 读取 skill 的 `templates/constraints/time_constraints_template.md`，写入 `[书名]/记忆系统/constraints/time_constraints.md`
       - 读取 skill 的 `templates/constraints/narrative_threads_template.md`，写入 `[书名]/记忆系统/constraints/narrative_threads.md`

    d) 填入已有设定信息（修改的是 `[书名]/记忆系统/` 下的副本）：
       - story_bible.md：书名/题材/世界观简版/字数目标/每章字数
       - character_arcs.md：已有角色初始状态（如有）
       - plot_timeline.md：主线事件/支线（如有）
       - chekhovs_gun.md：初始伏笔（如有）

8.3 【标记】待办事项到进度看板：
    - 添加任务：[ ] 05_03完成人物设计后更新 character_arcs.md
    - 添加任务：[ ] 05_04完成大纲后更新 plot_timeline.md
    - 保存：[书名]/设定/进度看板.md

8.4 【输出】记忆系统初始化完成报告：
    ```
    ✅ 记忆系统已初始化：[书名]/记忆系统/
    
    【故事圣经】
       ✓ story_bible.md（故事圣经模板）
       ✓ character_arcs.md（人物弧线模板）
       ✓ plot_timeline.md（剧情时间线模板）
       ✓ chekhovs_gun.md（伏笔清单模板）
    
    【强约束系统】
       ✓ cheat_constraints.md（金手指叙事约束）
       ✓ time_constraints.md（时间约束追踪）
       ✓ narrative_threads.md（叙事线监控）
    
    【下一步操作】
    1. 使用05_03模块设计人物后，更新 character_arcs.md
    2. 使用05_04模块生成大纲后，更新 plot_timeline.md
    3. 05_06（单章写作）会持续维护这些文档
    4. 强约束系统每章写作后自动检查（可在设定中补充金手指/时间线/叙事线信息）
    ```

9. 完成初始化流程
```

**输入信息**：
- 题材类型（玄幻/都市/言情/科幻/悬疑/历史等）
- 核心情绪标签（热血/甜蜜/悬疑/复仇/逆袭/救赎等）
- 创作目的（自娱/出版/网文连载）
- 目标读者群
- 预期篇幅（短篇/中篇/长篇/超长篇）
- 记忆系统：自动初始化（默认启用）

**输出**：
- 书名候选列表（直接反馈）
- 立项定位单：`[书名]/设定/[书名]_立项定位单.md`
- 进度看板：`[书名]/设定/进度看板.md`
- 记忆系统初始化报告（如选择初始化）
- 记忆系统目录：`[书名]/记忆系统/`（如选择初始化）

**记忆系统初始化说明**：
1. 记忆系统默认启用，在项目初始化时自动创建
2. 初始化后，05_06模块（单章写作）会自动维护记忆系统
3. 05_06模块优先使用记忆系统；如因特殊原因未初始化，自动降级到文本追踪表

**后续集成点**：
- 05_03模块（人物设计）：完成后自动同步到 [书名]/记忆系统/bible/character_arcs.md
- 05_04模块（大纲生成）：完成后自动同步到 [书名]/记忆系统/bible/plot_timeline.md
- 05_05模块（黄金开篇）：生成后自动创建首章摘要
- 05_06模块（单章写作）：自动更新4个圣经文档 + 单章摘要 + 阶段压缩
