# 规则集：赛博-技术流

> 硬约束规则集。硬件拓扑、算力因果、能源消耗的量化管理。
> 适用于需要硬核技术感的近未来赛博/科幻写作场景。

---

## 1. 元信息

```yaml
名称: 赛博-技术流
适用流派: 赛博朋克 / 硬科幻 / 黑客流 / 技术流网文
约束强度: 严格
简介: 强制硬件代价描写的严格规则集。每次算力消耗必须写物理反馈，损坏不可自动修复，跨区移动必须明确交通工具和耗时。
```

## 2. 能力因果法则

能力使用必须遵循完整逻辑链：

```
侦测 → 寻找跳板/漏洞 → 算力推演 → 夺取权限
```

**禁止行为**：
- ❌ 瞬间秒破——即使算力碾压也需要时间
- ❌ 跳过中间环节的直接获胜
- ✅ 破解高级目标必须设定并描写推演倒计时

## 3. 代价显性法则

| 能力级别 | 典型代价 | 必须描写的生理/环境反馈 |
|:--------|:---------|:----------------------|
| 小 | 普通运算 / 日常任务 | 轻微的氟化液温度上升 |
| 中 | 深度骇入 / 大规模并发 | 连续操作超 4 小时：手抖 / 出汗 / 低血糖 / 氟化液沸腾 |
| 大 | 硬件满载 / 超频突破 | 显卡啸叫 / 电压跌落 / 核心温度逼近物理熔毁阈值 |

## 4. 空间连续法则

| 移动方式 | 耗时基准 | 备注 |
|:---------|:---------|:-----|
| 下层区内部（步行）| 10-30 分钟 | — |
| 下层区 → 云端区 | 30-45 分钟 | 过境隧道/货运通道 |
| 跨国数据传输 | 跳板延迟叠加 | 每跳增加 50-200 ms |
| 物理移动到远程节点 | 取决于交通工具 + 跳板网络延迟 | 两项累加 |

## 5. 因果不可逆法则

- **死亡**：绝对不可逆。
- **硬件损毁**：烧毁的显卡/节点在未明确描写修复之前，保持损毁状态。
- **数据泄露**：一旦被获取，不可撤回。只能通过后续行动抑制后果。
- **位置暴露**：物理坐标暴露后，必须在合理时间窗口内出现追兵或撤离描写。

## 6. 资源约束追踪

**资源池字段**：
```yaml
- VRAM 总池 / 当前占用
- 计算卡数量 / 存活 / 烧毁 / 降频
- 能源供给（电网/电池/其他）
- 网络节点 / 跳板机数量
```

**实时工况字段**：
```yaml
- 核心温度（阵列平均 / 最高单卡 / 熔毁阈值）
- 冷却系统状态（液位 / 泵速 / 室温）
- 当前电网负荷 / 熔断风险
```
