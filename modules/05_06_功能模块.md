# 功能模块 6：单章写作（子代理写作·v9.0）

> **功能编号**：6
> **功能定位**：主代理纯调度——组装写作上下文后委托独立子代理完成章节创作。主代理只负责存文件+字数脚本校验，写作、审核、修复全部由子代理闭环。
> **架构版本**：v9.0 — 主代理从"写手"转为"编辑"，不再亲自写作。

---

## 一、核心特性

| 特性 | 说明 |
|------|------|
| **子代理写作** | 主代理读取记忆系统 → 组装写作 brief → delegate_task 委托独立子代理创作 |
| **四段式结构** | 子代理按开篇/铺垫/高潮/收尾结构创作 |
| **字数控制** | 子代理内控字数，主代理脚本校验；不达标→退回重写 |
| **记忆系统集成** | 写作子代理自行读取记忆文档（bible/arcs/timeline/chekhovs_gun/summaries/volumes） |

---

## 二、执行流程

### 步骤0：写前预检（05_24）

> 同旧版，不变。确认 05_24 写前预检已完成。若未执行 → 先加载 `modules/05_24_功能模块.md` 完成预检再回到本步骤。

### 步骤1：主代理判定章节类型

主代理快速读取以下关键文件，判定本章类型（不组装全量上下文）：

1. 章节规划表中本章的「章节类型」「爽点类型」「出场角色」「场景」「冲突目标」
2. chekhovs_gun.md 中状态为 🔄（准备回收）的伏笔

判定结果：

| 类型 | 特征 |
|------|------|
| 🔥 高潮章 | 爽点释放/大战/关键转折 |
| 📜 伏笔回收章 | 计划回收 ≥1 个远古伏笔 |
| 🌊 过渡章 | 日常/铺垫/支线展开 |
| 📦 标准章 | 常规推进（默认） |

### 步骤2：主代理读取字数配置 + 蓝图

1. 读取 `[书名]/设定/[书名]_立项定位单.md` 的「每章字数」字段 → 字下限=要求值，字上限=要求值+500
2. 未读到 → 默认 2000-2500
3. 同时检查 `story_bible.md` 的「每章字数」字段作为备用。冲突时以立项定位单为准，自动纠正 bible

### 步骤3：主代理启动写作子代理

通过 `delegate_task` 启动写作子代理（`toolsets:["file"]`）。

**写作子代理需读取的文件清单**（主代理给出绝对路径，子代理自行读取，不嵌入 context）：

**P0 必读（所有章节）**：
- `[书名]/记忆系统/bible/story_bible.md` → 当前状态快照 + 上一阶段梗概
- `[书名]/记忆系统/bible/character_arcs.md` → 全部角色的当前状态
- `[书名]/记忆系统/bible/plot_timeline.md` → 最近 20 章事件因果链
- `[书名]/记忆系统/bible/chekhovs_gun.md` → 全部活跃伏笔
- `[书名]/记忆系统/bible/character_database.md` → 已出场角色基础信息
- `[书名]/设定/[书名]_章节规划表.md` → 本章的出场角色/场景/冲突目标
- `templates/单章结构模板.md`（skill 目录绝对路径）

**P1 上下文（标准章+高潮章+伏笔回收章加载，过渡章仅最近3章）**：
- `[书名]/记忆系统/summaries/ch_{N-5}～{N-1}_summary.md` → 最近 5 章摘要
- `[书名]/记忆系统/volumes/volume_*.md` → 最近 5 个卷压缩（排除 _template.md）

**P2 按章节类型加配**：

| 类型 | 加配文件 |
|------|---------|
| 🔥 高潮章 | 额外 5 章摘要（共 10 章）+ 涉及角色的完整弧线 + 最近 2 个 volume |
| 📜 伏笔回收章 | 计划回收伏笔的首现章摘要 + 上次推进章摘要 |
| 🌊 过渡章 | P1 降为最近 3 章摘要 + 仅最近 1 个 volume；跳过伏笔详细审查 |

**P3 参考资源（按需，skill 目录绝对路径）**：

| 条件 | 加载资源 |
|------|---------|
| 章节类型 = 开篇冲突 且 ≤ 第5章 | `references/黄金开篇技巧.md` |
| 爽点类型字段 ≠ 空 | `references/爽文创作理论.md` + `references/爽点设计策略.md` |
| 高潮爆发/大高潮 + 战斗冲突 | `references/打斗场景创作指南.md` |
| 对话/对谈为主 | `references/风格技巧详解.md` |
| 冲突反转 | `references/爽文创作理论.md` |

**写作子代理 context 模板**：

```
goal: 创作小说《{书名}》第{N}章「{章节名}」

context: 中文创作。你是一位专业网文作家，需要创作第{N}章正文。

【章节信息】
- 章号：{N}
- 章名：{章节名}
- 字数范围：{字下限}-{字上限}字（中文汉字计数）
- 章节类型：{章节类型}（{类型说明}）
- 出场角色：{角色列表}
- 场景：{场景描述}
- 冲突目标：{冲突目标}

【写作要求】
1. 按四段式结构创作：开篇(10-15%) → 铺垫(20-30%) → 高潮(40-50%) → 收尾(15-20%)
2. 参考单章结构模板（templates/单章结构模板.md）
3. 字数控制在 {字下限}-{字上限} 范围内
4. 风格：{风格描述/默认"与前面章节保持一致"}

【必读文件】（请按以下顺序读取，均为绝对路径）
{P0 文件列表}
{P1 文件列表}
{P2 加配文件列表（如有）}
{P3 参考资源列表（如有）}

【输出格式】
返回纯文本，第一行为章节标题，第二行起为正文。严禁元数据、Markdown 标记、作者旁白。

【字数内控】
写完后自行估算中文字数。如估算不达标（<{字下限}），在返回前自行扩写至达标。
```

**⚠️ 占位符替换**：主代理在调用 delegate_task 前，必须将上述模板中所有 `{...}` 占位符替换为实际值，文件路径使用绝对路径。

### 步骤4：主代理接收 → 保存 → 字数检查

1. 接收子代理返回的正文文本
2. 写入 `[书名]/正文/第{N}章_{章节名}.txt`（纯文本，严禁元数据/Markdown）
3. 执行字数脚本检查：
   ```
   python -c "import re,sys; t=open(sys.argv[1], encoding='utf-8').read(); print(len(re.findall(r'[\u4e00-\u9fff]', t)))" "[书名]/正文/第{N}章_{章节名}.txt"
   ```
4. 字数 < 字下限 → ❌ 退回重写。主代理将字数反馈传给同一写作子代理（重新 delegate_task，context 中包含"上一版字数不足，需扩写至{字下限}字以上"）
5. 字数 > 字上限 → ❌ 退回精简（同上）
6. 字下限 ≤ 字数 ≤ 字上限 → ✅ 合格

### 步骤5：移交写后流水线

字数通过后 → **立即加载 `modules/05_26_功能模块.md` 并执行其全部步骤（步骤1-9），不得跳过。**

> 05_06 自身不执行记忆系统维护、不标记任何 .done 文件、不生成摘要。全部由 05_26 负责。
> 详细的圣经更新字段与写法以 05_26 步骤3 为唯一权威定义。

## 三、正文文件格式规范

同旧版，不变：

```
【文件名格式】
第XXX章_章节名.txt

【文件内容格式】
第一行：章节标题
第二行起：纯正文内容
```

- **严禁**在文件开头添加元数据
- **严禁**使用 Markdown 格式标记
- 文件必须是纯文本格式（流水线完成后的元数据块除外，由 05_26 步骤9 追加）

## 四、批量写章节流程

逐章执行完整流水线，每章独立闭环：

```
for i in range(N):
    章节编号 = 当前章节 + i
    步骤0：写前预检
    步骤1：判定章节类型
    步骤2：读字数配置+蓝图
    步骤3：启动写作子代理
    步骤4：接收→保存→字数检查（不达标退回重写）
    步骤5：移交 05_26 写后流水线（审核+修复+记忆维护）
    → 确认本章通过后，再进入下一章
```

⚠️ **禁止一次性写完所有章节再统一审核。**

## 五、与旧版的主要差异

| 环节 | v8.x（旧） | v9.0（新） |
|------|-----------|-----------|
| 写作执行者 | 主代理 | 独立子代理（delegate_task） |
| 上下文组装 | 主代理逐文件阅读后内化 | 主代理组装文件清单，子代理自行读取 |
| 写作连续性 | 依赖主代理单次会话内的"肌肉记忆" | 跨会话无差异（主代理和新子代理起点相同） |
| 记忆维护 | 主代理在 05_26 步骤3 执行 | 不变，主代理仍负责记忆维护（编辑职责） |
| 审核 | 7 个子代理，返回 PASS/FAIL | 不变（05_26 负责，v9.0 改为返回完整报告） |
| 修复 | 独立修复子代理 | 不变（05_26 负责，v9.0 修复子代理只读审核报告） |

## 七、注意事项

1. **占位符必须替换**：步骤3 的 context 模板中所有 `{...}` 必须在调用 delegate_task 前替换为实际值
2. **文件路径用绝对路径**：子代理的独立上下文中相对路径不可靠，必须给出绝对路径
3. **字数检查不可跳过**：脚本校验，不依赖 LLM 计数
4. **退回重写要传反馈**：字数不达标时，重新 delegate_task 的 context 中须包含具体字数差距
5. **写作子代理只返回正文**：不要返回元数据、解释、分析
6. **05_06 不标记 .done**：所有 .done 标记由 05_26 统一管理
