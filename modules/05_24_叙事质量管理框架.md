### 24. 叙事质量管理框架（Narrative Quality Control Framework）

**功能**：通用化的写前预检 → 写后复核(三阶) → 约束追踪 → 参数链条存档的完整质量控制工作流。支持通过可替换规则集适配不同写作流派（赛博/修仙/都市/西幻/历史/悬疑等）。

**定位**：框架层工具。不绑定任何世界观名词，只绑定叙事逻辑维度。与功能 17（连贯性检查）互补——功能 17 侧重内容一致性，本框架侧重叙事品质的量化管控。

**触发方式**：
- **写前**: `预检` / `准备写下一章` / `检查状态`
- **写后**: `复核` / `检查第X章` / `强约束`
- **初始化规则集**: `规则集` / `加载规则集`

---

## 架构概览

```
叙事质量管理框架
├── [核] 三阶工作流（写前预检 → 写后复核 → 链条存档）
├── [核] 五维审查维度模板（可替换）
│   ├── ① 能力因果法则
│   ├── ② 代价显性法则
│   ├── ③ 空间连续法则
│   ├── ④ 因果不可逆法则
│   └── ⑤ 资源约束追踪
├── [扩展] 规则集系统（`references/rulesets/`）
│   ├── _template.md          ← 自定义规则集模板
│   ├── 通用默认.md            ← 预设：维度齐全但约束宽松
│   ├── 赛博-技术流.md         ← 硬件守恒 / 算力因果
│   ├── 修仙-升级流.md
│   └── 都市-异能流.md
└── [输出] YAML 参数链条存档（每章末尾强制输出）
```

## 使用流程

### 初始化

首次使用，用户说 `规则集` 或 `加载规则集`：

```
检测到未加载规则集。请选择：

1. 通用默认（维度齐全，约束宽松——适合多数写作场景）
2. 赛博-技术流（硬件拓扑 / 算力因果 / 能源约束——硬核技术文）
3. 修仙-升级流（灵力循环 / 境界突破 / 天材地宝）
4. 都市-异能流（能量层级 / 异能显性消耗）
5. 自定义（从模板创建）

请回复编号，或输入流派名称。
```

选定后，框架读取对应规则集定义的五维审查标准，替代默认维度。

### 写前预检

用户说 `预检` 或 `准备写下一章`：

```
【章节状态核对】（基于 <规则集名称> 规则集）
- 当前时间点：
- 主角位置与状态：
- 资源约束（当前等级/可用资源）：
- 同步执行的进程：
- 待推进的伏笔：
```

任意一个字段无法填写 → 不准开始写正文。

### 写后复核（三阶）

用户说 `复核` 或 `检查第X章`：

#### 一阶：硬性检查（一条不通过则不发布）

| # | 检查项 | 检查方法 | 通过标准 |
|:-:|:-------|:---------|:---------|
| 1 | **字数达标** | 字数统计 | 章节 ≥ 约定字数（默认 3000） |
| 2 | **章末完整** | 检查章节结束标记 | 结束标记之后无内容 |
| 3 | **无元叙事** | 全文搜索"第一章""本章"等关键词 | 不应出现 |
| 4 | **文件名一致** | 文件名 vs 标题 | 完全一致 |

#### 二阶：五维规则审查（基于当前规则集）

| # | 维度 | 检查内容 | 危险信号 |
|:-:|:-----|:---------|:---------|
| 5 | **能力因果法则** | 力量/能力的使用是否遵循完整逻辑链 | 缺少中间步骤，或结果与能力不符 |
| 6 | **代价显性法则** | 每次使用能力是否支付了可感知代价 | 代价缺失/代价描写不足 |
| 7 | **空间连续法则** | 跨区域移动是否有合理耗时 | 无过程直接到达 |
| 8 | **因果不可逆法则** | 死亡/损坏状态是否被不合理翻转 | 无代价复活/修复 |
| 9 | **资源约束追踪** | 资源消耗是否与剧情匹配 | 资源无端恢复/超规格使用 |

#### 三阶：约束检查（叙事健康度）

| # | 维度 | 检查内容 | 数据来源 |
|:-:|:-----|:---------|:---------|
| 10 | **金手指代劳率** | 金手指/外挂解决了多少问题，主角自身成长解决了多少 | `constraints/cheat.md` |
| 11 | **时间流逝感** | 近几章时间流速是否合理 | `constraints/time.md` |
| 12 | **叙事线预期** | 哪些伏笔被推进，哪些被遗忘 | `constraints/threads.md` |

### 链条存档

复核通过后，在章节末尾输出 YAML 格式参数链条。模板见下。

---

## YAML 参数链条模板（通用版）

```yaml
---
[1. 时空常量锚点]
当前世界时间:
本章时间流逝:
距离下一关键节点倒计时:

[2. 主角状态]
坐标位置:
生理/状态指标:
可用资源:

[3. 力量体系与资源池]   ← 规则集自定义字段
当前等级/阶段:
资源约束拓扑: （规则集定义）
实时工况: （规则集定义）

[4. 空间与情报拓扑]
已掌控的路径/跳板:
当前领域/权限:
暴露/威胁风险指数:

[5. 剧情因果锁（Plot Flags - 绝对不可逆）]
已触发不可逆事件:
当前未回收异常/伏笔:
---
```

规则集可自定义第 3 区块的专有字段，其余区块通用。

## 规则集规范

每个规则集是一个 Markdown 文件，存放在 `references/rulesets/` 下，需包含以下 6 节。详细模板见 `references/rulesets/_template.md`。

| 节 | 内容 |
|:---|:-----|
| 1. 元信息 | 名称 / 适用流派 / 约束强度 / 简介 |
| 2. 能力因果法则 | 能力使用必须遵循的逻辑链步骤定义 |
| 3. 代价显性法则 | 不同等级能力对应的典型代价 |
| 4. 空间连续法则 | 本流派典型移动方式及合理耗时参考 |
| 5. 因果不可逆法则 | 不可逆事件的定义范围 |
| 6. 资源约束追踪 | 资源池字段清单 + 实时工况字段 |

## 与现有模块的对接

| 模块 | 对接方式 |
|:----|:---------|
| 功能 03 实时状态追踪 | 本框架的写前预检字段读取 `[书名]/记忆系统/` 中的状态记录 |
| 功能 17 连贯性检查 | 二阶段复核完成后，可选触发功能 17 做深度一致性检查 |
| 功能 22 强约束系统 | 本框架的三阶约束检查直接调用功能 22 的数据文档 |
| 记忆系统 | 约束检查结果记入 `[书名]/记忆系统/constraints/` |

## 质量检查

使用本框架完成复核后，自查：

- [ ] 写前预检的 4+1 个字段全部填写完整？
- [ ] 一阶硬性检查全部通过？
- [ ] 二阶五维审查是否有标记异常项？
- [ ] 三阶约束检查是否已更新约束文档？
- [ ] 链条存档已输出？（写后才输出，写前不输出）
- [ ] 规则集已加载？（首次使用必须加载，否则默认使用通用默认）
