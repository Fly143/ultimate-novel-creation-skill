# 质量保证指南

## 概述

质量是小说成功的基石。通过系统的质量控制和检查机制，确保作品在内容、文字、节奏等方面都达到较高水准。

本指南整合了**质量控制清单**、**生成前后检查机制**、**质量问题修正策略**、**连贯性维护工具**四大核心模块，提供系统化的质量保证方案。

---

## 一、质量控制清单

### 内容质量

**情节连贯性**
- [ ] 情节是否连贯
- [ ] 逻辑是否自洽
- [ ] 时间线是否正确
- [ ] 因果关系是否合理

**人物一致性**
- [ ] 人物性格是否一致
- [ ] 说话风格是否统一
- [ ] 行为模式是否合理
- [ ] 实力成长是否符合设定

**设定统一性**
- [ ] 世界观设定是否统一
- [ ] 力量体系是否稳定
- [ ] 前后是否有矛盾
- [ ] 伏笔是否回收

**细节准确性**
- [ ] 事实是否准确
- [ ] 数据是否正确
- [ ] 称呼是否一致
- [ ] 错别字是否修正

---

### 文字质量

**语言流畅性**
- [ ] 语言是否流畅
- [ ] 句子结构是否合理
- [ ] 是否有重复啰嗦
- [ ] 是否有语病

**描写生动性**
- [ ] 描写是否生动
- [ ] 意象是否具体
- [ ] 比喻是否恰当
- [ ] 细节是否丰富

**对话自然性**
- [ ] 对话是否自然
- [ ] 符合人物性格
- [ ] 推动情节发展
- [ ] 不生硬说教

**文字规范性**
- [ ] 标点符号是否正确
- [ ] 段落划分是否合理
- [ ] 格式是否统一
- [ ] 错别字是否修正

---

### 节奏质量

**节奏合理性**
- [ ] 节奏是否合理
- [ ] 张弛有度
- [ ] 不拖沓不仓促
- [ ] 转换自然

**爽点到位性**
- [ ] 爽点是否到位
- [ ] 铺垫是否充分
- [ ] 爆发是否有力
- [ ] 余韵是否悠长

- [ ] 不夸张不做作
- [ ] 读者有共鸣
- [ ] 推进自然

## 二、章节生成前检查清单

### 1. 信息回顾

**回顾前3章关键信息**：
- [ ] 主角当前状态：实力、位置、心理状态、人际关系
- [ ] 当前情节进度：处于哪个阶段（开篇/发展/高潮），主要冲突是什么
- [ ] 已埋伏笔：有哪些伏笔已经埋下，需要在近期提及或回收

**回顾方法**：
- 快速浏览前3章的结尾部分
- 提取关键信息：人物状态、当前冲突、待解决事项
- 记录：主角现在在哪里？在做什么？面临什么挑战？

**状态快照模板**：
```
=== 第X章状态快照 ===

【主角状态】
- 姓名：
- 实力等级：
- 当前位置：
- 心理状态：
- 人际关系：

【当前阶段】
- 阶段：[开篇/发展/高潮]
- 主要冲突：
- 待解决事项：

【待回收伏笔】
- 伏笔1：[内容] - 埋下章节：[X章] - 预期回收：[X章]
- 伏笔2：[内容] - 埋下章节：[X章] - 预期回收：[X章]
```

---

### 2. 情节类型确认

**情节类型定义**：

| 类型 | 功能 | 内容 | 节奏 |
|------|------|------|------|
| **推进型** | 推动剧情发展 | 新事件、新挑战、新信息 | 快 |
| **展开型** | 展开人物关系、世界观 | 人物互动、场景描写 | 中 |
| **爽点型** | 释放爽点，满足期待 | 反击、打脸、逆袭、突破 | 快 |
| **过渡型** | 连接高潮，调整节奏 | 主角休整、实力提升 | 慢 |

**避免重复**：
- [ ] 前一章是什么类型？
- [ ] 前两章是什么类型？
- [ ] 本章是否与前两章类型不同？
- [ ] 如果连续使用同一类型，是否已调整侧重点？

**建议节奏**：
```
推进 → 展开 → 爽点 → 过渡 → 推进
（避免连续3章使用同一类型）
```

---

### 3. 爽点类型确认

**爽点类型清单**：
- 扮猪吃虎
- 越级碾压
- 身世揭秘
- 反派打脸
- 资源垄断
- 荣耀加身
- 复仇闭环
- 其他

**避免单调**：
- [ ] 本章是否有爽点？
- [ ] 爽点类型是什么？
- [ ] 前两章的爽点类型是什么？
- [ ] 本章爽点类型是否与之前不同？

**建议频率**：
- 每5章内包含至少3种不同爽点类型
- 避免连续3章使用同一爽点类型

---

### 4. 字数目标确认

**字数要求**：
- 目标：1500-2000字
- 最低：1500字
- 理想：1700-1900字

**字数分配**：

| 部分 | 字数 | 内容 |
|------|------|------|
| 开头 | 100-200字 | 承接上章、抛出新问题 |
| 核心 | 1000-1500字 | 主要情节发展 |
| 结尾 | 200-300字 | 收束本章、引向下章 |

**检查清单**：
- [ ] 字数目标是否明确？
- [ ] 字数分配是否合理？
- [ ] 有哪些扩充方法可以使用？

## 三、章节生成后检查清单

### 1. 基础检查（每章必查）

**字数检查**：
- [ ] 字数是否≥1500字？
- [ ] 如果不足，哪些部分可以扩充？
- [ ] 优先扩充：心理描写、细节描写、对话

**情节连贯性检查**：
- [ ] 本章开头是否与上一章结尾衔接？
- [ ] 情节推进是否合理？
- [ ] 因果关系是否清晰？
- [ ] 时间线是否正确？

**人物一致性检查**：
- [ ] 主角性格是否一致？
- [ ] 反派性格是否一致？
- [ ] 配角行为是否符合设定？
- [ ] 人物关系是否有矛盾？

**爽点效果检查**：
- [ ] 爽点是否达到预期？
- [ ] 铺垫是否充分？
- [ ] 爆发是否有力？
- [ ] 反应是否真实？

---

### 2. 深度检查（每5章执行一次）

#### 跨章连贯性检查

**人物状态连续性**：
- [ ] 主角实力是否与前章衔接？（如：上章炼气3层，本章应保持或有合理提升）
- [ ] 主角位置是否合理过渡？（如：上章在宗门，本章不应无故出现在远方）
- [ ] 主角心理状态是否连贯？（如：上章愤怒，本章不应无故平静）
- [ ] 配角状态是否一致？（如：配角上章受伤，本章应有伤势表现）

**伏笔追踪检查**：
- [ ] 已埋伏笔是否记录在「创作状态追踪表」中
- [ ] 待回收伏笔是否按计划推进
- [ ] 新埋伏笔是否有明确回收计划
- [ ] 伏笔回收时机是否恰当（不宜过早或过晚）

**时间线验证**：
- [ ] 事件间隔是否合理（天/周/月）
- [ ] 时间描述是否一致（如"三天后"是否真的过了三天）
- [ ] 季节/时间流逝是否合理
- [ ] 事件持续时间是否符合逻辑

**设定一致性检查**：
- [ ] 力量体系是否冲突（如：筑基期不该有元婴期能力）
- [ ] 世界规则是否冲突（如：已设定的规则是否被违反）
- [ ] 势力关系是否冲突（如：敌对势力不应突然友好）
- [ ] 资源设定是否冲突（如：已消耗的资源不应再出现）

---

#### 人物一致性深度检查

**主角一致性**：
- [ ] 性格是否保持一致？（隐忍/桀骜/扮猪吃虎等）
- [ ] 说话风格是否统一？
- [ ] 行为模式是否合理？
- [ ] 成长路径是否按计划？

**反派一致性**：
- [ ] 性格是否前后一致？
- [ ] 实力变化是否合理？
- [ ] 与主角的关系是否连贯？
- [ ] 崩溃/臣服的原因是否充分？

**配角一致性**：
- [ ] 每个配角的性格是否稳定？
- [ ] 对主角的态度变化是否自然？
- [ ] 行为是否符合设定？
- [ ] 出场是否有意义？

**人物关系一致性**：
- [ ] 主角与反派的关系是否连贯？
- [ ] 主角与配角的关系是否合理发展？
- [ ] 配角之间的关系是否一致？
- [ ] 势力关系是否符合设定？

---

#### 情节逻辑深度检查

**时间线**：
- [ ] 章节之间的时间顺序是否正确？
- [ ] 事件的发生时间是否合理？
- [ ] 有无时间跳跃的矛盾？
- [ ] 事件的持续时间是否合理？

**因果关系**：
- [ ] 前因后果是否清晰？
- [ ] 事件的触发是否合理？
- [ ] 后果是否与原因匹配？
- [ ] 有无无中生有的情节？

**冲突升级**：
- [ ] 冲突的升级是否自然？
- [ ] 冲突的解决是否合理？
- [ ] 支线冲突与主线冲突是否关联？
- [ ] 冲突的激烈程度是否得当？

**逻辑漏洞**：
- [ ] 有无情节矛盾？
- [ ] 有无无法解释的事件？
- [ ] 有无逻辑不通的转折？
- [ ] 有无前后矛盾的设定？

---

#### 设定符合度深度检查

**实力等级**：

- [ ] 反派的实力变化是否合理？
- [ ] 配角的实力是否一致？

- [ ] 金手指的使用限制是否一致？

- [ ] 势力关系是否符合设定？
- [ ] 场景描写是否一致？

**资源**：

- [ ] 资源的用途是否一致？

#### 伏笔回收深度检查

**伏笔记录**：
- [ ] 已记录所有已埋下的伏笔
- [ ] 已标记伏笔的类型和重要性

## 四、质量问题修正策略

### 小问题修正（口误、小矛盾）

**问题类型**：
- 口误、笔误
- 小矛盾（前后细节不一致）
- 轻微时间线偏差

**修正方法**：
1. 用旁白稍作解释
   示例："其实，那已经是三天前的事了。"
   
2. 用对话自我纠正
   示例："等等，我记错了，应该是..."

**修正时机**：立即修正，无需延迟

---

### 中等问题修正（情节偏差）

**问题类型**：
- 情节偏差（与预期走向不符）
- 人物性格轻微不一致
- 实力等级小波动

**修正方法**：
1. 通过情节发展解释
   示例：让主角经历一个事件，解释其性格变化的原因
   
2. 用回忆补充说明
   示例："林萧想起了之前的经历，那让他明白了..."

**修正时机**：1-2章内完成

---

### 大问题修正（核心设定矛盾）

**问题类型**：
- 核心设定矛盾
- 主线情节错误
- 重大人物崩塌

**修正方法**：
1. 暂停创作，与用户沟通
2. 确认修正方向
3. 可能需要重写相关章节

**修正时机**：立即与用户沟通

---

### 问题修正分级策略表

| 问题级别 | 问题类型 | 修正方法 | 示例 |
|----------|----------|----------|------|
| 小问题 | 口误、数值偏差 | 对话或旁白自然修正 | 第10章修为写成"筑基"，第11章对话："上次突破筑基后..." |
| 中问题 | 情节偏差 | 1-2章内通过情节修正 | 主角提前获得神器→第12章增加"神器认主条件"限制 |
| 大问题 | 设定冲突 | 立即启动「设定回溯」流程 | 性格突变→与用户确认修正方向 |

## 五、连贯性维护工具

### 1. 人物卡片维护

**人物卡片格式**：
```
人物姓名：
  - 性格：[关键词]
  - 当前状态：[实力/地位/关系]
  - 核心目标：[目标]
  - 关键关系：[与主角/其他角色的关系]
  - 重要特征：[标志性特征]
  - 发展轨迹：[已发生的变化]
  - 待发展：[未来可能的发展方向]
```

**维护方法**：
- 每次角色出场后更新状态
- 记录角色的关键变化
- 标注角色的重要时刻
- 定期回顾人物卡片

---

### 2. 时间线维护

**时间线格式**：
```
第X章（事件名称）：
  - 发生时间：[具体时间]
  - 关键人物：[参与人物]
  - 事件影响：[对后续的影响]
  - 关联事件：[相关章节]
```

**维护方法**：
- 按章节顺序记录重要事件
- 标注事件之间的时间间隔
- 记录事件的因果关系
- 定期检查时间线是否连贯

---

### 3. 伏笔追踪表

**伏笔追踪格式**：
```
伏笔名称：
  - 埋下章节：[章节号]
  - 伏笔类型：[人物/情节/设定]
  - 重要程度：[高/中/低]
  - 预期回收：[预期章节]
  - 状态：[未回收/已回收/进行中]
```

**维护方法**：
- 及时记录新埋下的伏笔
- 标记伏笔的回收时机
- 定期检查是否有被遗忘的伏笔
- 优先回收重要伏笔

---

### 4. 设定对照表

**设定对照格式**：
```
设定类别：
  - 原始设定：[前四阶段的设定]
  - 当前状态：[实际执行的情况]
  - 差异：[不一致的地方]
  - 修正计划：[如何修正]
```

**维护方法**：
- 定期与原始设定对比
- 记录设定执行中的偏差
- 规划修正方案
- 在后续章节中自然修正

---

## 六、质量保证流程

### 初稿阶段

**写作时自检**
- 写完一章后立即检查
- 重点检查：错别字、语病、格式
- 确保基本质量

**记录问题**
- 发现问题记录下来
- 不影响阅读的可以后续修正
- 关键问题立即修正

---

### 完稿阶段

**全面检查**
- 全文通读
- 使用质量控制清单逐项检查
- 标记所有问题

**分类整理**
- 内容问题
- 文字问题
- 节奏问题

**制定修正计划**
- 优先级排序
- 规划修正时间
- 明确修正目标

---

### 修正阶段

**内容修正**
- 修复逻辑漏洞
- 调整情节不合理之处
- 统一设定和人物

**文字修正**
- 润色语言
- 优化描写
- 改进对话

**节奏修正**
- 调整章节节奏
- 优化爽点分布
- 增强悬念

---

### 完善阶段

**最终检查**
- 再次使用质量控制清单
- 确保所有问题已解决
- 检查是否引入新问题

## 八、常见质量问题及解决方案

### 内容问题

**逻辑漏洞**
- 表现：因果关系不合理，时间线混乱
- 解决：建立时间线，检查因果关系，修复漏洞

**人物OOC**
- 表现：人物行为与性格不符
- 解决：建立人物卡片，定期回顾人物设定

**设定矛盾**
- 表现：前后设定不一致
- 解决：建立设定文档，严格遵守设定

---

### 文字问题

**语言啰嗦**
- 表现：重复表达，冗余信息多
- 解决：精简文字，删除冗余

**描写空洞**
- 表现：缺乏细节，意象模糊
- 解决：增加细节描写，使用具体意象

**对话生硬**
- 表现：不符合人物，像说教
- 解决：根据人物性格调整对话，推动情节

---

### 节奏问题

**节奏拖沓**
- 表现：推进缓慢，读者不耐烦
- 解决：精简冗余内容，加快节奏

**节奏仓促**
- 表现：推进太快，铺垫不足
- 解决：增加铺垫，放慢节奏

**爽点不到位**
- 表现：铺垫不足，爆发无力
- 解决：增加铺垫，强化爆发
