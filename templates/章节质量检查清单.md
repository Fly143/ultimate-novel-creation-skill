# 章节质量检查清单

本清单用于确保每章生成内容的质量稳定性。请在章节生成前后严格执行。

---

## 一、每章生成前必须确认

### 1. 信息回顾

**回顾前3章关键信息**：
- [ ] 主角当前状态：实力、位置、心理状态、人际关系
- [ ] 当前情节进度：处于哪个阶段（开篇/发展/高潮），主要冲突是什么
- [ ] 已埋伏笔：有哪些伏笔已经埋下，需要在近期提及或回收

**回顾方法**：
- 快速浏览前3章的结尾部分
- 提取关键信息：人物状态、当前冲突、待解决事项
- 记录：主角现在在哪里？在做什么？面临什么挑战？

**状态快照模板**：
```
=== 第X章状态快照 ===

【主角状态】
- 姓名：
- 实力等级：
- 当前位置：
- 心理状态：
- 人际关系：

【当前阶段】
- 阶段：[开篇/发展/高潮]
- 主要冲突：
- 待解决事项：

【待回收伏笔】
- 伏笔1：[内容] - 埋下章节：[X章] - 预期回收：[X章]
- 伏笔2：[内容] - 埋下章节：[X章] - 预期回收：[X章]
```

---

### 2. 情节类型确认

**情节类型定义**：

| 类型 | 功能 | 内容 | 节奏 |
|------|------|------|------|
| **推进型** | 推动剧情发展 | 新事件、新挑战、新信息 | 快 |
| **展开型** | 展开人物关系、世界观 | 人物互动、场景描写 | 中 |
| **爽点型** | 释放爽点，满足期待 | 反击、打脸、逆袭、突破 | 快 |
| **过渡型** | 连接高潮，调整节奏 | 主角休整、实力提升 | 慢 |

**避免重复**：
- [ ] 前一章是什么类型？
- [ ] 前两章是什么类型？
- [ ] 本章是否与前两章类型不同？
- [ ] 如果连续使用同一类型，是否已调整侧重点？

**建议节奏**：
```
推进 → 展开 → 爽点 → 过渡 → 推进
（避免连续3章使用同一类型）
```

---

### 3. 爽点类型确认

**爽点类型清单**：
- 扮猪吃虎
- 越级碾压
- 身世揭秘
- 反派打脸
- 资源垄断
- 荣耀加身
- 复仇闭环
- 其他

**避免单调**：
- [ ] 本章是否有爽点？
- [ ] 爽点类型是什么？
- [ ] 前两章的爽点类型是什么？
- [ ] 本章爽点类型是否与之前不同？

**建议频率**：
- 每5章内包含至少3种不同爽点类型
- 避免连续3章使用同一爽点类型

---

### 4. 字数目标确认

**字数要求**：
- 目标：1500-2000字
- 最低：1500字
- 理想：1700-1900字

**字数分配**：
| 部分 | 字数 | 内容 |
|------|------|------|
| 开头 | 100-200字 | 承接上章、抛出新问题 |
| 核心 | 1000-1500字 | 主要情节发展 |
| 结尾 | 200-300字 | 收束本章、引向下章 |

**检查清单**：
- [ ] 字数目标是否明确？
- [ ] 字数分配是否合理？
- [ ] 有哪些扩充方法可以使用？

---

## 二、每章生成后必须检查

### 1. 基础检查

**字数检查**：
- [ ] 字数是否≥1500字？
- [ ] 如果不足，哪些部分可以扩充？
- [ ] 优先扩充：心理描写、细节描写、对话

**情节连贯性检查**：
- [ ] 本章开头是否与上一章结尾衔接？
- [ ] 情节推进是否合理？
- [ ] 因果关系是否清晰？
- [ ] 时间线是否正确？

**人物一致性检查**：
- [ ] 主角性格是否一致？
- [ ] 反派性格是否一致？
- [ ] 配角行为是否符合设定？
- [ ] 人物关系是否有矛盾？

**爽点效果检查**：
- [ ] 爽点是否达到预期？
- [ ] 铺垫是否充分？
- [ ] 爆发是否有力？
- [ ] 反应是否真实？

---

### 2. 深度检查（每5章执行一次）

#### 跨章连贯性检查

**人物状态连续性**：
- [ ] 主角实力是否与前章衔接？（如：上章炼气3层，本章应保持或有合理提升）
- [ ] 主角位置是否合理过渡？（如：上章在宗门，本章不应无故出现在远方）
- [ ] 主角心理状态是否连贯？（如：上章愤怒，本章不应无故平静）
- [ ] 配角状态是否一致？（如：配角上章受伤，本章应有伤势表现）

**伏笔追踪检查**：
- [ ] 已埋伏笔是否记录在「创作状态追踪表」中
- [ ] 待回收伏笔是否按计划推进
- [ ] 新埋伏笔是否有明确回收计划
- [ ] 伏笔回收时机是否恰当（不宜过早或过晚）

**时间线验证**：
- [ ] 事件间隔是否合理（天/周/月）
- [ ] 时间描述是否一致（如"三天后"是否真的过了三天）
- [ ] 季节/时间流逝是否合理
- [ ] 事件持续时间是否符合逻辑

**设定一致性检查**：
- [ ] 力量体系是否冲突（如：筑基期不该有元婴期能力）
- [ ] 世界规则是否冲突（如：已设定的规则是否被违反）
- [ ] 势力关系是否冲突（如：敌对势力不应突然友好）
- [ ] 资源设定是否冲突（如：已消耗的资源不应再出现）

**问题修正策略**：

| 问题级别 | 问题类型 | 修正方法 | 示例 |
|----------|----------|----------|------|
| 小问题 | 口误、数值偏差 | 对话或旁白自然修正 | 第10章修为写成"筑基"，第11章对话："上次突破筑基后..." |
| 中问题 | 情节偏差 | 1-2章内通过情节修正 | 主角提前获得神器→第12章增加"神器认主条件"限制 |
| 大问题 | 设定冲突 | 立即启动「设定回溯」流程 | 性格突变→与用户确认修正方向 |

**连贯性维护工具**：
- 创作状态追踪表：实时记录人物状态、当前情节、已埋伏笔
- 时间线记录表：记录关键事件的时间和持续时间
- 设定一致性表：记录力量体系、世界规则、势力关系

#### 人物一致性深度检查

**主角一致性**：
- [ ] 性格是否保持一致？（隐忍/桀骜/扮猪吃虎等）
- [ ] 说话风格是否统一？
- [ ] 行为模式是否合理？
- [ ] 成长路径是否按计划？

**反派一致性**：
- [ ] 性格是否前后一致？
- [ ] 实力变化是否合理？
- [ ] 与主角的关系是否连贯？
- [ ] 崩溃/臣服的原因是否充分？

**配角一致性**：
- [ ] 每个配角的性格是否稳定？
- [ ] 对主角的态度变化是否自然？
- [ ] 行为是否符合设定？
- [ ] 出场是否有意义？

**人物关系一致性**：
- [ ] 主角与反派的关系是否连贯？
- [ ] 主角与配角的关系是否合理发展？
- [ ] 配角之间的关系是否一致？
- [ ] 势力关系是否符合设定？

#### 情节逻辑深度检查

**时间线**：
- [ ] 章节之间的时间顺序是否正确？
- [ ] 事件的发生时间是否合理？
- [ ] 有无时间跳跃的矛盾？
- [ ] 事件的持续时间是否合理？

**因果关系**：
- [ ] 前因后果是否清晰？
- [ ] 事件的触发是否合理？
- [ ] 后果是否与原因匹配？
- [ ] 有无无中生有的情节？

**冲突升级**：
- [ ] 冲突的升级是否自然？
- [ ] 冲突的解决是否合理？
- [ ] 支线冲突与主线冲突是否关联？
- [ ] 冲突的激烈程度是否得当？

**逻辑漏洞**：
- [ ] 有无情节矛盾？
- [ ] 有无无法解释的事件？
- [ ] 有无逻辑不通的转折？
- [ ] 有无前后矛盾的设定？

#### 设定符合度深度检查

**实力等级**：
- [ ] 主角的实力成长是否符合设定？
- [ ] 反派的实力变化是否合理？
- [ ] 配角的实力是否一致？
- [ ] 实力差距是否保持？

**金手指**：
- [ ] 金手指的能力是否符合设定？
- [ ] 金手指的解锁时机是否合理？
- [ ] 金手指的使用限制是否一致？
- [ ] 金手指的升级路径是否按计划？

**世界观**：
- [ ] 世界规则是否保持统一？
- [ ] 势力关系是否符合设定？
- [ ] 场景描写是否一致？
- [ ] 特殊设定是否保持？

**资源**：
- [ ] 资源的获取是否合理？
- [ ] 资源的消耗是否符合逻辑？
- [ ] 资源的数量是否连续？
- [ ] 资源的用途是否一致？

#### 伏笔回收深度检查

**伏笔记录**：
- [ ] 已记录所有已埋下的伏笔
- [ ] 已标记伏笔的类型和重要性
- [ ] 已预估伏笔的回收时机
- [ ] 已标记待回收伏笔

**伏笔回收**：
- [ ] 伏笔是否被遗忘？
- [ ] 伏笔回收时机是否恰当？
- [ ] 伏笔揭示是否自然？
- [ ] 伏笔回收是否有力？

**新伏笔**：
- [ ] 新埋下的伏笔是否有意义？
- [ ] 新伏笔是否与前文矛盾？
- [ ] 新伏笔是否有利于后续情节？
- [ ] 新伏笔是否已记录？

### 3. 创新深度检查（每10章执行一次）

#### 情节类型多样性

**检查清单**：
- [ ] 最近10章使用了哪些情节类型？
- [ ] 是否有过度使用的类型？
- [ ] 情节类型是否多样化？
- [ ] 是否需要调整后续章节的情节类型？

**统计方法**：
```
| 情节类型 | 使用次数 | 占比 |
|----------|----------|------|
| 推进型 | X次 | XX% |
| 展开型 | X次 | XX% |
| 爽点型 | X次 | XX% |
| 过渡型 | X次 | XX% |
```

**建议比例**：
- 推进型：40-50%
- 展开型：20-30%
- 爽点型：20-30%
- 过渡型：10-20%

#### 人物成长深度

**检查清单**：
- [ ] 主角的成长是否只是实力提升？
- [ ] 主角是否有心理成长？
- [ ] 主角的世界观是否有所变化？
- [ ] 主角的性格是否有发展？

**成长维度**：
| 维度 | 检查要点 |
|------|----------|
| 实力成长 | 等级提升、技能解锁、装备获取 |
| 心理成长 | 心态变化、认知提升、情感成熟 |
| 关系成长 | 人际扩展、信任建立、羁绊加深 |
| 世界观成长 | 认知扩大、价值观变化、目标调整 |

#### 主题体现

**检查清单**：

- [ ] 主题是否在情节中得到体现？
- [ ] 主题是否在人物成长中得到体现？
- [ ] 主题是否有所升华？

#### 创新元素

**检查清单**：
- [ ] 是否有反套路情节？
- [ ] 是否有意外转折？
- [ ] 是否有深度冲突？
- [ ] 是否有独特设定？

## 三、质量问题修正

### 小问题修正（口误、小矛盾）

**示例**：
- 第10章：主角姓名写成"林晓"
- 第11章修正：用对话或旁白稍作说明
- 修正方式："林萧，不，是林晓..."（自我纠正）

**修正原则**：
- 不影响主要情节
- 修正要自然
- 不要打断阅读流畅度

### 中等问题修正（情节偏差）

**示例**：
- 第10章：主角实力提升过快，超出了设定
- 第11-12章：通过情节发展进行修正

**修正方式**：
- 第11章：主角遭遇挑战，实力提升的效果不如预期
- 第12章：主角发现问题，重新调整修炼方法

**修正原则**：
- 在1-2章内修正
- 通过情节发展自然修正
- 不要直接修改已生成内容

### 大问题修正（核心设定矛盾）

**示例**：
- 第10-15章：主角性格突然大变，与设定不符

**修正方式**：
- 与用户沟通，确认是否需要重新调整方向

**修正原则**：
- 与用户沟通确认
- 可能需要重新调整方向
- 不要擅自修改
