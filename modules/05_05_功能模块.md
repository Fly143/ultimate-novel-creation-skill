### 5. 黄金开篇创作 

**功能**：设计开篇冲突、建立强烈期待，可选创建首章摘要到记忆系统
**参考资源**：`references/黄金开篇技巧.md`、`references/首战必胜逻辑链.md`、`references/风格把控方法.md`
**模板**：`templates/黄金开篇模板.md`
**记忆系统**：支持创建首章摘要到 [书名]/记忆系统/summaries/ch_001_summary.md

**执行流程（强制按顺序执行）**：

```
=== 黄金开篇创作阶段 ===
1. 读取章节规划表，确认第1章规划
2. 读取 `references/风格把控方法.md`，确定全书文风基调
3. 设计开篇冲突和期待建立方式
4. 【逐章创作黄金三章】按"写一章→完整写后流水线→确保 PASS→再下一章"的闭环执行（中途不停顿、不询问）：
   a. 创作第1章正文 → 保存第001章_章节名.txt
   b. 立即对该章执行写后流水线（字数校验→主代理逻辑自检→圣经更新×5→摘要→7个子代理审核→修复），全部 PASS 后才进入第2章
   c. 创作第2章正文 → 同样完整写后流水线闭环，PASS 后进入第3章
   d. 创作第3章正文 → 同样完整写后流水线闭环
   ⚠️ 严禁先一口气写完第1-3章再统一审核。每一章必须独立走完 写作→审核→修复 闭环，确认无问题后才写下一章。
   ⚠️ 写后流水线（05_26）已包含圣经更新×5+摘要（步骤3），本模块不再重复执行圣经更新。
   ⚠️ 降级模式触发链（自动，无需用户干预）：05_05步骤4 → 调用05_26 → 05_26前置检查检测记忆系统未初始化 → 05_26降级模式(调用05_18工作流A) → 初始化完成 → 05_26步骤3正常执行
5. 更新章节规划表状态

=== 记忆系统同步 ===
6. 【记忆系统检查】黄金开篇的写后流水线（步骤4 的 05_26 闭环）已自动完成圣经更新×5+摘要。
   本模块不重复执行圣经更新。仅在记忆系统未初始化时（首次新开），在步骤4 之前由 05_26 降级模式自动初始化。
   - 如需手动检查记忆系统状态，使用功能18 工作流B（`看记忆`）

7. 完成黄金开篇流程
```

**输入信息**：
- 开篇类型（冲突式/悬念式/反差式等）
- 核心设定展示需求
- 目标平台（可选）
- 是否同步到记忆系统（可选）

**输出**：
- 开篇正文（第1-3章）：`[书名]/正文/第001章_章节名.txt` 等
- 首章同步报告（如记忆系统已初始化）

**记忆系统集成说明**：

| 功能 | 实现方式 | 触发时机 |
|------|---------|---------|
| **圣经+摘要更新** | 05_26 写后流水线步骤3（5圣经+摘要） | 每章创作后（05_26 闭环） |
| **质量自检** | 05_26 写后流水线 7 子代理审核（A-G） | 每章创作后（05_26 闭环） |

**注意事项**：
1. v5.0 文风一致性由 LLM 自行判断（对比前期章节），无需外部工具
2. 如果后续需要调整文风，通过05_18模块手动编辑圣经文档指导方向
