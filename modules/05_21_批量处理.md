# 功能模块 21：批量处理

**功能**：一次生成多章内容，提高创作效率。

---

## 批量写章节

### 使用方式

```
输入：写接下来5章
输入：批量写第10-15章
输入：一口气写10章
```

### 执行流程

```
1. 确认章节范围（起始章-结束章）
2. 加载记忆系统（如已初始化）
3. 循环执行单章写作流程：
   for 章节号 in 范围:
     - 生成章节记忆包
     - 创作章节内容
     - 保存章节文件
     - 更新记忆系统
     - 执行质量检查
4. 生成批量创作报告
```

### 输出报告

```
=== 批量创作完成报告 ===

【创作统计】
- 章节范围：第10-15章（共6章）
- 总字数：18,520字
- 平均每章：3,087字
- 耗时：约45分钟

【质量检查】
- 风格偏离度：平均12.3%（良好）
- 人物一致性：全部通过
- 伏笔回收：2个
- 新增伏笔：3个

【文件清单】
- 第010章_xxx.txt
- 第011章_xxx.txt
- ...
```

## 批量生成设定

### 使用方式

```
输入：批量生成10个配角人设
输入：生成5条支线剧情
输入：设计3个反派角色
```

### 执行流程

1. 确认生成数量和类型
2. 加载世界观和主线设定
3. 批量生成并保存
4. 生成批量报告

---

## 批量诊断

### 使用方式

```
输入：诊断第1-20章的节奏
输入：检查所有章节的人物一致性
```

### 执行流程

1. 确认诊断范围
2. 逐章执行诊断
3. 生成综合诊断报告

---

## 注意事项

- 批量创作时每章仍会执行记忆系统更新
- 建议单次批量不超过10章，避免上下文过载
- 批量完成后建议人工抽查质量
