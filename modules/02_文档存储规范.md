# 文档存储规范


**所有产出内容必须自动保存为文件，严禁仅输出在对话中。**

### 设定类内容 → `.md` 格式

| 内容类型 | 文件命名 | 说明 |
|----------|----------|------|
| 项目立项 | `[书名]_立项定位单.md` | 包含题材、情绪标签、目标读者、篇幅规划 |
| 世界观设定 | `[书名]_世界观设定.md` | 世界框架、力量体系、规则、势力分布 |
| 人物设定 | `[书名]_人设卡.md` | 主角、配角、反派完整人设（可多个文件） |
| 人物关系 | `[书名]_人物关系图谱.md` | 角色关系、冲突关系、情感关系 |
| 情节大纲 | `[书名]_情节大纲.md` | 主线结构、章节规划、爽点分布 |
| 章节规划表 | `[书名]_章节规划表.md` | 逐章内容概要、字数、状态、爽点类型 |
| 黄金开篇 | `[书名]_开篇设定.md` | 开篇冲突、期待建立、核心设定展示 |
| 创作状态追踪 | `创作状态追踪表.md` | 记忆系统报告、人物状态、剧情进展 |
| 进度看板 | `进度看板.md` | 项目进度、任务状态、待办事项 |

**存储规则**：
- 以书名为根目录，如 `我的修仙小说/`
- 所有设定文件存放在 `设定/` 子目录下
- 文件命名统一使用 `[书名]_内容类型.md` 格式（创作状态追踪表和进度看板除外）

### 记忆系统内容 → `.md` 格式（v5.0 自然语言记忆）

| 内容类型 | 文件命名 | 说明 |
|----------|----------|------|
| 故事圣经 — 世界观/阶段 | `记忆系统/bible/story_bible.md` | 项目概况、世界观简版、阶段划分、状态快照 |
| 故事圣经 — 人物弧线 | `记忆系统/bible/character_arcs.md` | 人物演化轨迹（追踪变化，非静止设定） |
| 故事圣经 — 剧情时间线 | `记忆系统/bible/plot_timeline.md` | 主线/支线事件因果链 |
| 故事圣经 — 伏笔清单 | `记忆系统/bible/chekhovs_gun.md` | 已埋/待收/已收伏笔追踪 |
| 角色数据库 | `记忆系统/bible/character_database.md` | 角色详细档案（性格/能力/关系/成长轨迹） |
| 单章摘要 | `记忆系统/summaries/ch_NNN_summary.md` | 每章一档，记录事件/人物变化/伏笔 |
| 阶段压缩 | `记忆系统/phases/phase_N_chX-Y.md` | 每10章压缩块，用于跨会话回忆 |
| 卷压缩 | `记忆系统/volumes/volume_N_chX-Y.md` | 每50章卷压缩，最高层级记忆缓存 |

**v5.0**：
- ✅ **LLM 直接读写**：LLM 打开文件就能理解内容，无需解析
- **存储规则**：记忆系统统一放在 `记忆系统/` 目录下（与 `设定/` 平级）

### 正文类内容 → `.txt` 格式

| 内容类型 | 文件命名 | 说明 |
|----------|----------|------|
| 章节正文 | `第001章_章节名.txt` | 纯文本格式，方便导入写作软件 |
| 第002章 | `第002章_章节名.txt` | 依次递增编号 |
| ... | ... | ... |

**存储规则**：
- 正文文件存放在 `正文/` 子目录下
- 文件命名格式：`第XXX章_章节名.txt`（编号三位数，不足补零）
- 每章独立一个文件，便于管理和修改

### 章节审计元数据（每章末尾必追加）

写后流水线全部完成后，在章节 `.txt` 文件末尾追加以下元数据块：

```
## 元数据
字符数：[N]
完成步骤：wordcount/logic_selfcheck/bible/summary/gates/constraints/review/rhythm/quality/coherence/logic_causal
审核评分：[分数]/100
审核时间：[YYYY-MM-DD]
```

**规则**：
- 元数据块位于章节正文末尾，`---` 分隔线之后
- 下次加载本章时，先检查末尾元数据的「完成步骤」字段
- 缺失的步骤自动触发「补缺」流程补齐
- 跨会话恢复时以此判断上一章是否完整走完

### 诊断报告 → `.md` 格式

| 内容类型 | 文件命名 | 说明 |
|----------|----------|------|
| 节奏诊断报告 | `第NNN章_节奏诊断报告.md` | 节奏分析、问题诊断、优化建议 |
| 商业可行性评估 | `商业可行性评估报告.md` | 市场分析、竞品对比、风险评估 |
| 创新深度检查 | `创新深度检查报告.md` | 创新点分析、差异化评估 |

**存储规则**：
- 诊断报告存放在 `设定/` 目录下
- 文件命名包含对应章节或时间点，便于追踪

### 执行要求

1. **生成即保存**：每完成一个设定模块或章节正文，立即调用文件保存工具
2. **增量追加**：章节正文按编号递增，不覆盖已有文件
3. **进度追踪**：保存后更新进度看板，记录已完成的内容
4. **文件交付**：在任务结束或用户要求时，通过 `finish` 工具交付所有生成的文件
5. **记忆系统统一管理**：所有记忆系统文件统一存放在 `记忆系统/` 目录下，与 `设定/` 平级
