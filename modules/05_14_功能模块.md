# 功能14 创新深度检查

## 子代理调用规范

本模块必须作为独立子代理运行，使用 `delegate_task` 调用。

### 调用模板

```
delegate_task(
    goal="对小说作品进行全面的创新深度检查，评估情节、人物、主题、设定四个维度的创新性与原创度，生成检查报告并提供改进建议",
    context="""
    项目目录：[项目根目录]
    必需文件：
    [书名]/设定/[书名]_世界观设定.md
    [书名]/设定/[书名]_人设卡.md
    [书名]/设定/[书名]_情节大纲.md
    [书名]/设定/[书名]_世界观设定.md（覆盖世界观）
    [书名]/正文/（所有已写章节）
    [书名]/设定/创新深度检查报告.md（如已存在则覆盖）

检查维度：
1. 情节创新：是否避免常见套路，是否有新颖性和意外性
2. 人物创新：是否立体多面，成长是否自然，关系是否复杂
3. 主题创新：是否有深度，价值观是否独特，是否有社会意义
4. 设定创新：世界观是否独特，规则是否自洽，是否有创意
""",
    toolsets=['file']
)
```

### 输出要求

- 报告保存至：`[书名]/设定/创新深度检查报告.md`
- 包含四个维度的详细评估
- 每个维度给出创新等级评分（1-10）
- 提供具体的改进建议和替换方案
- 标注需要重点改进的段落

---

## 检查流程

### 步骤一：确认检查范围

与用户确认本次检查的维度范围：

1. **情节创新** — 检查故事走向、冲突设置、转折安排
2. **人物创新** — 检查角色塑造、成长弧线、关系网络
3. **主题创新** — 检查核心主题、价值观表达、思想深度
4. **设定创新** — 检查世界观、规则体系、背景设定

默认：全维度检查。用户可指定单维度或组合维度。

---

### 步骤二：读取相关文件

读取以下文件作为检查依据：

| 文件 | 用途 |
|------|------|
| `[书名]_情节大纲.md` | 情节走向、转折点、高潮设置 |
| `[书名]_人设卡.md` | 角色塑造、性格设计、关系网络 |
| `[书名]_世界观设定.md` | 核心主题、价值观、思想表达 + 世界规则、体系设定、背景框架 |
| `正文/` 目录下所有文件 | 实际内容的创新表现 |

---

### 步骤三：多维度检查

#### 3.1 情节创新检查

评估要点：

- **避免套路**：是否存在过度使用的桥段（如失忆、替身、重生复仇模板化）
- **新颖性**：核心冲突是否独特，是否能在同类型中脱颖而出
- **意外性**：转折是否出人意料但合乎逻辑
- **节奏创新**：叙事节奏是否有变化，是否避免单一模式
- **因果创新**：事件的因果链是否有独创性

检查清单：

- [ ] 是否存在明显的同类作品套路？
- [ ] 核心冲突是否有独特切入点？
- [ ] 高潮桥段是否出人意料？
- [ ] 结局是否避免烂尾/强行团圆？
- [ ] 叙事视角是否有创新尝试？

创新等级评分（1-10）：
- 1-3分：严重套路化，缺乏新意
- 4-6分：有一定创新，但仍有明显模仿痕迹
- 7-8分：创新突出，能给人耳目一新的感觉
- 9-10分：极具开创性，可能引领新的创作方向

#### 3.2 人物创新检查

评估要点：

- **维度丰富性**：角色是否多面立体，而非单一标签
- **成长自然度**：角色变化是否有合理铺垫，是否突兀
- **关系复杂性**：人物关系是否有层次，而非简单的善恶对立
- **动机独特性**：角色行为动机是否有独到之处
- **配角存在感**：配角是否有独立人格，而非工具人

检查清单：

- [ ] 主角是否有3个以上鲜明性格维度？
- [ ] 角色成长是否有足够的情节支撑？
- [ ] 反派是否有令人信服的动机？
- [ ] 女性角色是否独立有深度（避免花瓶化）？
- [ ] 配角是否有自己的故事线？
- [ ] 人物关系是否超越简单的二元对立？

创新等级评分（1-10）：
- 1-3分：角色扁平，严重脸谱化
- 4-6分：有一定深度，但仍能看到明显原型
- 7-8分：角色立体鲜活，令人印象深刻
- 9-10分：角色塑造具有文学经典品质

#### 3.3 主题创新检查

评估要点：

- **深度挖掘**：主题是否有深层探讨，而非浮于表面
- **价值独特性**：传递的价值观是否有独到见解
- **社会关联**：是否能引发对现实问题的思考
- **多义性**：主题是否有多重解读空间
- **时代性**：是否能回应当代读者的精神需求

检查清单：

- [ ] 核心主题是否超越简单的善恶判断？
- [ ] 是否有对人性的深度探讨？
- [ ] 价值观表达是否有独立见解？
- [ ] 是否能引发读者的深层思考？
- [ ] 主题与故事是否有机融合（而非说教）？

创新等级评分（1-10）：
- 1-3分：主题空洞或过于说教
- 4-6分：有一定深度，但缺乏独到见解
- 7-8分：主题深刻，具有启发性
- 9-10分：具有哲学深度，可能影响读者世界观

#### 3.4 设定创新检查

评估要点：

- **世界观独特性**：是否构建了独特的世界，而非照搬现有体系
- **规则自洽性**：设定的规则是否逻辑自洽
- **细节创意**：世界观细节是否有巧思
- **体系创新**：力量体系/社会体系是否有新意
- **文化融合**：是否融合了多元文化元素

检查清单：

- [ ] 世界观是否有独特的标志性元素？
- [ ] 力量/魔法体系是否有独创规则？
- [ ] 社会结构是否有新意？
- [ ] 设定是否自洽（无逻辑矛盾）？
- [ ] 设定是否服务于故事（而非为设定而设定）？

创新等级评分（1-10）：
- 1-3分：设定粗糙或照搬现有作品
- 4-6分：有一定创意，但缺乏系统性
- 7-8分：设定完整独特，有标志性元素
- 9-10分：设定极具开创性，足以成为经典

### 步骤五：提供改进建议

针对检查中发现的问题，提供具体可执行的改进方案：

#### 改进方案格式

```
问题：[具体问题描述]
位置：[章节名/段落位置]
严重程度：高/中/低

改进方案：
1. [方案一：具体修改建议]
2. [方案二：替换/新增内容建议]
3. [方案三：结构调整建议]

参考示例：[优秀的创新案例参考]
```

#### 常见创新提升策略

**情节创新提升：**
- 反转经典套路：找到常见桥段，从反面重新构建
- 多线叙事：增加叙事层次，避免单一主线
- 时间跳跃：打破线性叙事，增加信息差
- 视角转换：尝试非常规叙事视角

**人物创新提升：**
- 缺陷设计：为主角设计不完美的一面
- 动机复杂化：让角色的动机既有合理性又有矛盾
- 关系灰色地带：增加人物关系的模糊性和复杂度
- 成长代价：让每次成长都有明确的代价

**主题创新提升：**
- 多元视角：从不同立场探讨同一主题
- 悖论设计：引入道德困境，避免简单答案
- 隐喻体系：建立贯穿全书的象征系统
- 时代对话：与当代社会议题建立联系

**设定创新提升：**
- 文化混搭：融合多种文化元素创造新体系
- 规则反转：挑战读者的既有认知
- 细节深化：在设定细节中注入独特创意
- 代价体系：为超自然元素设定明确的代价规则

---

## 注意事项

1. **检查必须独立运行**：通过 delegate_task 调用，确保检查的客观性
2. **保留原始内容**：改进建议不直接修改原文，而是提供替换方案
3. **尊重风格**：创新建议必须在保持作品整体风格的前提下提出
4. **平衡创新与可读性**：创新不能以牺牲可读性为代价
5. **标注具体位置**：所有问题必须标注具体章节和段落，方便定位
