# 强制执行协议

**本技能加载后，你必须严格执行以下协议，不得跳过或简化：**

### 🚨 首次交互必须执行

当用户触发本技能时（提到小说创作、写故事、构思情节、设计人物、规划章节、搭建世界观等），**执行以下智能识别流程**：

#### 第一步：意图识别

1. **读取智能启动模块**：`read_file("modules/00_智能启动协议.md")`
2. **分析用户输入**，判断意图类型：
   - **继续创作意图**：提到章节号、小说名称、"继续写"、"下一章"等
   - **单个功能意图**：提到具体功能名称（书名、人设、大纲、诊断等）
   - **新开小说意图**：说"写小说"、"创作故事"或描述创作想法

#### 第二步：根据意图执行

**继续创作意图** → 直接进入场景B，先检查记忆系统：
```
检测到您想继续创作，请提供：

1. 小说名称（如有）
2. 当前进度（写到第几章）
3. 本次目标（继续写下一章/诊断节奏/修改设定/其他）

[系统自动检测 记忆系统/ 的存在]
如果 记忆系统/ 存在：
→ 自动读取 bible/ 4文档 + 最新 phases + 最近 5 summaries
→ 组装 L4 上下文包，快速恢复创作状态
如果 记忆系统/ 不存在：
→ 提示用户：未检测到记忆系统，建议先初始化（功能18）
```

**单个功能意图** → 直接进入场景C：
```
检测到您需要[功能名称]功能，正在加载...
[直接执行对应功能模块]
```

**新开小说意图或无明确意图** → 显示简化引导：
```
您好！我是小说创作助手，请问：

A. 新开小说 - 从零开始创作
B. 继续创作 - 接着写已有项目
C. 单个功能 - 使用特定功能

请回复 A/B/C，或直接告诉我您的需求。
```

#### 第三步：等待用户响应

在用户做出选择或提供信息前，不得执行任何创作任务。

**禁止行为**：
- ❌ 直接开始创作内容
- ❌ 直接输出小说设定或章节
- ❌ 跳过意图识别直接进入功能
- ❌ 当用户意图明确时仍强制显示完整菜单

## 🔴 方案一：阻断式检查点

**每个多步骤流程中，在关键步骤后必须插入 STOP 标记。后续步骤开头必须引用检查上一阶段是否完成。**

### 写前检查点
当执行「继续」/「下一章」时，按以下顺序执行，**每步完成后确认再继续下一步**：

```
步骤A：搜索项目记忆系统
  → STOP：确认搜索到项目（未找到 → 转场景A新开项目）

步骤B：读取记忆系统（bible×4 + summaries + phases）
  → STOP：确认4个圣经文档全部读取完成（缺文档 → 自动初始化补齐）

步骤C：执行写前预检（加载05_24叙事质量管理框架）
  → STOP：确认预检通过（缺字段 → 阻断并自愈修复，重新预检）
      未通过不得进入写作阶段

步骤D：加载05_06单章写作模块 → 开始写作
```

### 写后检查点
章节保存后，按以下顺序执行，**上一步未完成不得进入下一步**：

```
步骤1：字数强制检查（05_06步骤6）
  → STOP：字数不达标（<下限或>上限）→ 必须扩写/精简重来
  ✅ 字数达标 → 标记 .done_wordcount

步骤2：更新4个圣经文档（story_bible + character_arcs + plot_timeline + chekhovs_gun）
  → STOP：确认4个全部更新完成
  ✅ 全部更新 → 标记 .done_bible

步骤3：生成单章摘要（写入 summaries/ch_NNN_summary.md）
  → STOP：确认摘要文件已创建
  ✅ 已创建 → 标记 .done_summary

步骤4：五道生成门禁（引用/一致性/蓝图/未知实体/描写）
  → STOP：任意门禁不通过 → 修复后重新检查
  ✅ 全部通过 → 标记 .done_gates

步骤5：强约束检查（金手指/时间/叙事线）
  → STOP：确认3个约束文件已更新
  ✅ 全部更新 → 标记 .done_constraints

步骤6：风格与质量自检
  → STOP：自检通过
  ✅ 通过 → 标记 .done_selfcheck

步骤7：触发读者审稿（加载05_25_读者审稿.md）
  → 审稿完成 → 标记 .done_review
```

## 🔴 方案二：拆细快捷命令

将旧的大流程「继续」拆分为两组细粒度的触发词。LLM 看到"继续"就只做写作部分，写完后必须等用户说「写后」或自动触发才做流水线。

### 写作命令
| 触发词 | 执行范围 |
|--------|---------|
| 「继续」/「下一章」 | 仅执行：上下文恢复 → 写前预检 → 单章写作 → 字数检查 → 保存文件 |
| 「第X章」 | 同上，定位到第X章续写/重写 |
| 「重写第X章」 | 覆盖指定章节，其余同上 |

### 写后命令
| 触发词 | 执行范围 |
|--------|---------|
| 「写后」 / 「收尾」 | 执行完整写后流水线：圣经更新→摘要→门禁→约束→自检→读者审稿 |
| 「半程收尾」 | 仅执行：圣经更新 + 摘要 + 字数确认（快速模式，用于用户催更） |
| 「补缺」 | 检查并补全写后流水线中遗漏的步骤（根据 .done 标记判定） |

### 单功能命令（不变）
| 触发词 | 加载模块 |
|--------|---------|
| 「书名」「人设」「大纲」「爽点」「战斗」「诊断」「导出」等 | 对应功能模块 |

### 智能触发规则
- 用户说「继续」写完后 → 自动询问「要执行写后流水线吗？」
- 用户说「继续」后紧接着说「下一章」→ 连续写作模式，每3章自动触发一次写后流水线
- 用户长时间不说话 → 在写后流水线完成后自动停在「读者审稿」步骤等待确认

## 🔴 方案三：文件锁（.done 标记状态墙）

**在每个步骤完成后，在 `[书名]/.done/` 目录下创建标记文件。后续步骤必须先检查对应标记存在才能执行。**

### 标记文件规范

```
[书名]/.done/
├── .done_wordcount       # 字数检查通过
├── .done_bible           # 4个圣经文档更新完成
├── .done_summary         # 单章摘要生成完成
├── .done_gates           # 五道门禁全部通过
├── .done_constraints     # 强约束检查完成
├── .done_selfcheck       # 风格自检通过
├── .done_review          # 读者审稿完成
├── .done_chapter_NNN     # 第NNN章完整走完
├── .done_phase_01        # 阶段1完成
├── .done_volume_01       # 卷1完成
```

### 检查规则

**步骤执行前必须检查前置标记：**
```
要执行步骤N → 检查 .done_step(N-1) 是否存在
  → 存在：继续执行
  → 不存在：提示「步骤N-1未完成」，自动调用步骤N-1
```

**写后流水线入口检查：**
```
用户说「写后」/自动触发写后 → 检查 .done_wordcount 是否存在
  → 存在：继续执行圣经更新
  → 不存在：提示「字数检查未通过，请先完成写作」
```

**跨会话恢复检查：**
```
新会话加载技能 → 检查 [书名]/.done/ 目录
  → 存在 .done_chapter_NNN → 上次已完整走完第NNN章，安全
  → 存在部分标记（如只有 .done_wordcount 无 .done_bible）
    → 提示「检测到上次写后流水线未完成，是否继续补齐？」
```

### 标记创建方式

```
步骤完成后 → 执行 touch [书名]/.done/.done_STEPNAME
```

### 标记自动清理

| 触发条件 | 清理内容 |
|---------|---------|
| 进入新一章写作 | 删除 .done_wordcount ~ .done_review（清空上一章的步骤标记） |
| 完成整章审核（.done_review 创建后） | 创建 .done_chapter_NNN 作为整章完成的锚点 |
| 进入新阶段（每10章） | 保留 .done_chapter_NNN，删除 .done_phase_N（用新的覆盖） |

## 🔄 每次功能调用必须执行

当用户选择某个功能模块后：
1. **加载模块**：读取对应功能模块文件
2. **读取对应参考资源**：按模块定义加载 `references/` 下的文件
3. **使用对应模板**：按模块定义加载 `templates/` 下的文件
4. **按流程执行**：严格遵循模块的「执行流程」步骤

### 💾 所有产出必须保存

- 设定类内容 → 保存为 `.md` 文件
- 正文类内容 → 保存为 `.txt` 文件
- **禁止仅输出在对话中不保存文件**

### 🔧 错误处理

当遇到错误时：
1. **模块缺失**：提示用户并提供替代方案
2. **路径错误**：尝试自动修复或降级到基础功能
3. **用户输入不明确**：显示简化引导而非报错

### ⚡ 快捷命令一览

| 分类 | 触发词 | 执行范围 |
|------|--------|---------|
| 写作 | 「继续」「下一章」「第X章」「重写第X章」 | 仅写作→字数检查，不含写后流水线 |
| 写后 | 「写后」「收尾」「半程收尾」「补缺」 | 执行/补齐写后流水线各步骤 |
| 功能 | 「书名」「人设」「大纲」「诊断」「爽点」「战斗」「导出」「看记忆」等 | 对应功能模块 |
| 管理 | 「菜单」「帮助」「改设定」「人物跑偏了」「规则集」 | 索引/管理功能 |

## 🛑 特殊规则：LLM 不按流程执行时的兜底

当 LLM 因自身局限跳过或漏执行某步骤时，以下规则生效：

1. **用户指出漏步** → 立即停止当前动作，执行 `补缺` 流程
2. **.done 标记检测到断裂** → 从断裂点自动回溯补齐，无需用户再次指示
3. **跨会话时.done目录不完整** → 以最新一章为准，补全缺失标记后继续
4. **「继续」后未触发写后流水线** → 自动在字数检查完成后询问「需要执行写后流水线吗？」
