### 5. 黄金开篇创作 

**功能**：设计开篇冲突、建立强烈期待，可选创建首章摘要到记忆系统
**参考资源**：`references/黄金开篇技巧.md`、`references/风格把控方法.md`
**模板**：`templates/黄金开篇模板.md`
**记忆系统**：支持创建首章摘要到 [书名]/记忆系统/summaries/ch_001_summary.md

**执行流程（强制按顺序执行）**：

```
=== 黄金开篇创作阶段 ===
1. 读取章节规划表，确认第1章规划
2. 读取 `references/风格把控方法.md`，确定全书文风基调
3. 设计开篇冲突和期待建立方式
4. 【逐章创作黄金三章】按"写一章→完整写后流水线→确保 PASS→再下一章"的闭环执行（中途不停顿、不询问）：
   a. 创作第1章正文 → 保存第001章_章节名.txt
   b. 立即对该章执行写后流水线（字数校验→圣经更新×4→摘要→6个子代理审核→修复），全部 PASS 后才进入第2章
   c. 创作第2章正文 → 同样完整写后流水线闭环，PASS 后进入第3章
   d. 创作第3章正文 → 同样完整写后流水线闭环
   ⚠️ 严禁先一口气写完第1-3章再统一审核。每一章必须独立走完 写作→审核→修复 闭环，确认无问题后才写下一章。
5. 保存正文：[书名]/正文/第001章_章节名.txt（依次递增）
6. 更新章节规划表状态

=== 记忆系统同步（如已初始化） ===
7. 【记忆系统检查】检查记忆系统是否已初始化：
   - 检查路径：[书名]/记忆系统/ 目录是否存在
   - 检查文件：[书名]/记忆系统/bible/story_bible.md 是否存在
   - 如果已初始化，执行步骤7.1-7.2；如果未初始化，跳转到步骤8

7.1 【创建首章摘要】将开篇内容写入单章摘要：
    - 读取刚生成的正文：[书名]/正文/第001章_*.txt
    - 按照 [书名]/记忆系统/summaries/_template.md 格式创建 ch_001_summary.md
    - 写入：[书名]/记忆系统/summaries/ch_001_summary.md
    - 同时更新 story_bible.md：
      * 读取 [书名]/记忆系统/bible/story_bible.md
      * 更新「当前状态快照」（填入第1章结尾场景和悬念）
      * 更新「当前已写章节」为 1

7.2 【输出】首章同步完成报告：
    ```
    ✅ 开篇信息已同步到记忆系统

    【已同步】
    ✓ ch_001_summary.md 已创建
    ✓ story_bible.md 状态快照已更新

    【下一步】
    1. 使用05_06模块继续单章写作
    2. 05_06会自动维护4个圣经文档 + 生成后续摘要
    ```

8. 完成黄金开篇流程
```

**输入信息**：
- 开篇类型（冲突式/悬念式/反差式等）
- 核心设定展示需求
- 目标平台（可选）
- 是否同步到记忆系统（可选）

**输出**：
- 开篇正文（第1-3章）：`[书名]/正文/第001章_章节名.txt` 等
- 首章同步报告（如记忆系统已初始化）

**记忆系统集成说明**：

| 功能 | 实现方式 | 触发时机 |
|------|---------|---------|
| **首章摘要创建** | LLM 直接写入 summaries/ch_001_summary.md | 完成第1章后 |
| **圣经快照更新** | LLM 更新 story_bible.md | 每章创作后（05_06模块） |
| **文风自检** | LLM 对比前一阶段压缩 | 每章创作后（05_06模块） |

**注意事项**：
1. v5.0 文风一致性由 LLM 自行判断（对比前期章节），无需外部工具
2. 如果后续需要调整文风，通过05_18模块手动编辑圣经文档指导方向
