# 强约束系统 — 设计方案（v6.7.0 规划）

> 状态：已实施（v6.7.0），设计已确认（2026-05-11）。
> 当前 v6.6.3 无此功能。实施时 bump → v6.7.0。

---

## 概述

新增**功能 23：强约束系统**，三个子系统合并在一个模块中：

1. **金手指发展约束** — 追踪主角金手指的解锁条件、使用代价、膨胀速度
2. **时间约束** — 追踪多条 deadline，倒计时提醒，交叉冲突检查
3. **多条叙事线监控** — 追踪各条叙事线的活跃度，冷落警告，覆盖度报告

三个子系统的共同特征：从现有记忆系统的「事后检查一致性」升级为「事前约束预防问题」。

## 子系统 1：金手指发展约束

### 追踪内容

| 字段 | 说明 | 示例 |
|------|------|------|
| 金手指名称 | 唯一标识 | 祖传玉佩 |
| 类型 | 系统/法宝/血脉/重生记忆等 | 法宝 |
| 当前等级 | 当前解锁阶段 | Lv.2（共7层） |
| 解锁条件 | 升到下一级需要满足什么 | 击杀3只筑基妖兽 |
| 使用代价 | 每次使用的限制 | 每日3次，过量则昏迷 |
| 冷却时间 | 两次使用间隔 | 1盏茶（约15分钟） |
| 上次升级章节 | 防止膨胀过快的锚点 | 第8章 |
| 最低升级间隔 | 相邻升级至少隔 N 章 | 5章 |

### 存储位置

`[书名]/记忆系统/constraints/cheat_constraints.md`

### 触发方式

- **初始化**：功能 1（项目初始化）或功能 2（世界观构建）时询问用户设置
- **自动检查**：每章写作后，检查本章是否涉及金手指使用/升级，是否违反约束
- **自动警告**：膨胀过快（距上次升级不足最低间隔）→ 主动警告
- **手动触发**：`金手指` / `检查金手指`

## 子系统 2：时间约束

### 追踪内容

| 字段 | 说明 | 示例 |
|------|------|------|
| deadline 名称 | 唯一标识 | 宗门大比 |
| 故事内截止时间 | 在故事世界中的日期/倒计时 | 7天后 |
| 当前故事时间 | 上次记录的故事内时间 | 开元历 327 年 3 月 12 日 |
| 截止章节预估 | 按当前节奏预计在第几章触发 | 第 15 章 |
| 紧迫度 | 🟢宽松 / 🟡临近 / 🔴过期 | 🟡 临近（差 3 章） |
| 关联叙事线 | 属于哪条叙事线 | 主线 |

### 存储位置

`[书名]/记忆系统/constraints/time_constraints.md`

### 触发方式

- **添加**：大纲阶段或用户随时手动添加（`添加deadline：宗门大比，7天后`）
- **自动更新**：每章写作后，更新当前故事时间，重新计算各 deadline 紧迫度
- **自动警告**：任何 deadline 变 🔴 → 主动警告用户
- **交叉检查**：两个 deadline 时间冲突 → 提示用户
- **手动触发**：`时间约束` / `deadline`

## 子系统 3：多条叙事线监控

### 追踪内容

| 字段 | 说明 | 示例 |
|------|------|------|
| 叙事线名称 | 唯一标识 | 苏晴暗线 |
| 主角/POV | 这条线的主要视角角色 | 苏晴 |
| 类型 | 主线/支线/暗线 | 支线 |
| 优先级权重 | 1-5，越高越不能冷落 | 4 |
| 当前状态 | 🟢活跃 / 🟡暂停 / ⚫完结 | 🟢 活跃 |
| 最后活跃章节 | 上次出现在哪一章 | 第 8 章 |
| 当前进度摘要 | 一句话概括最新进展 | 苏晴发现玉佩秘密，正调查来源 |
| 下个节点 | 这条线下一步的关键事件 | 苏晴找到玉佩原主后人（计划第 18 章） |

### 注意力评分

```
注意力分 = (当前章节号 - 最后活跃章节号) × 优先级权重
```

- 分 > 15：🟡 提醒用户该线被冷落
- 分 > 30：🔴 强烈建议插入场景
- 分 > 50：🚨 严重警告，线可能已被遗忘

### 交叉冲突检测

- 两条线是否在同一故事时间发生 → 是否有矛盾
- 两条线的角色是否应该相遇但未相遇 → 提示

### 覆盖度报告

每次阶段压缩时生成：各条线占总章数的比例，检测是否失衡。

### 存储位置

`[书名]/记忆系统/constraints/narrative_threads.md`

### 触发方式

- **初始化**：大纲阶段（功能 4）或用户随时手动添加
- **自动更新**：每章写作后更新各线状态
- **自动警告**：注意力评分超阈值 → 主动警告
- **手动触发**：`叙事线` / `支线监控`
