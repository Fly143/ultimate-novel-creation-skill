# 智能启动协议

**本模块优化了启动流程，能够智能识别用户意图，提供更流畅的交互体验。**

---

## 一、意图识别规则

### 1.1 直接进入场景B（继续创作）

**触发条件**（满足任一即可）：
- 用户提到具体章节号（如"写第5章"、"继续第10章"）
- 用户提到小说名称（如"继续写《xxx》"）
- 用户说"继续写"、"接着写"、"下一章"
- 用户上传了已有项目文件

**响应方式**：
```
检测到您想继续创作，正在加载项目...

[直接进入场景B流程，跳过A/B/C选择]
```

### 1.2 直接进入场景C（单个功能）

**触发条件**（满足任一即可）：
- 用户明确提到功能名称（如"生成书名"、"设计人设"、"诊断节奏"）
- 用户说"帮我xxx"且xxx是具体功能
- 用户提到"模板"、"大纲"、"人物关系"等关键词

**响应方式**：
```
检测到您需要[功能名称]功能...

[直接加载对应功能模块，跳过菜单选择]
```

### 1.3 进入场景A（新开小说）

**触发条件**：
- 用户说"写小说"、"创作故事"、"新开一本"
- 用户描述了创作想法但没有具体项目
- 其他未匹配到B或C的情况

**响应方式**：
```
检测到您想创作新小说，请选择创作模式：

[显示场景A的模式选择]
```

## 二、快捷入口

### 2.1 一键继续创作

**用户输入示例**：
- "继续"
- "接着写"
- "下一章"
- "写第X章"

**处理流程**：
1. 检查是否有最近的项目记录
2. 如果有，直接加载项目并进入单章写作
3. 如果没有，提示用户选择项目

### 2.2 快速功能调用

**用户输入示例**：
- "书名" → 直接进入书名生成功能
- "人设" / "人物" → 直接进入人物设计功能
- "大纲" → 直接进入情节大纲功能
- "诊断" / "节奏" → 直接进入节奏诊断功能
- "开篇" / "黄金开头" → 直接进入黄金开篇功能

---

## 三、上下文记忆

### 3.1 项目记录

**记录内容**：
- 最近使用的项目名称
- 最后创作的章节号
- 上次使用时间

**存储位置**：
- `[书名]/记忆系统/` 目录下（项目初始化时创建）

### 3.2 智能推荐

**推荐逻辑**：
- 如果有最近项目，优先推荐继续创作
- 如果用户经常使用某个功能，优先推荐该功能
- 根据时间间隔调整推荐（长时间未使用则重新引导）

## 四、优化后的启动流程

### 4.1 标准流程（无明确意图）

```
用户输入触发词 → 
读取本模块 → 
显示简化引导（3个选项）→ 
等待用户选择 → 
执行对应场景
```

### 4.2 智能流程（有明确意图）

```
用户输入触发词 → 
读取本模块 → 
意图识别 → 
  ├─ 继续创作意图 → 直接进入场景B
  ├─ 单个功能意图 → 直接进入场景C
  └─ 新开小说意图 → 显示场景A模式选择
执行对应流程
```

## 五、引导话术优化

### 5.1 简化版引导（推荐使用）

```
您好！我是小说创作助手，请问：

**A. 新开小说** - 从零开始创作
**B. 继续创作** - 接着写已有项目  
**C. 单个功能** - 使用特定功能

请回复 A/B/C，或直接告诉我您的需求。
```

### 5.2 智能识别后引导

**识别到继续创作**：
```
检测到您想继续创作，请提供：
1. 小说名称
2. 当前进度（写到第几章）
3. 本次目标（继续写下一章/诊断节奏/其他）
```

**识别到单个功能**：
```
检测到您需要[功能名称]功能，正在加载...
[直接执行功能，跳过菜单]
```

---

## 六、与原协议的兼容

### 6.1 降级机制

如果智能识别失败或用户输入不明确，自动降级到原版启动引导（01_启动引导.md）。

### 6.2 强制模式

用户可随时输入"菜单"或"帮助"强制显示完整功能菜单。
