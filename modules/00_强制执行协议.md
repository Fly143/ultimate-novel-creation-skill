# 强制执行协议

**本技能加载后，你必须严格执行以下协议，不得跳过或简化：**

### 🚨 首次交互必须执行

当用户触发本技能时（提到小说创作、写故事、构思情节、设计人物、规划章节、搭建世界观等），**执行以下智能识别流程**：

#### 第一步：意图识别

1. **读取智能启动模块**：`read_file("modules/00_智能启动协议.md")`
2. **分析用户输入**，判断意图类型：
   - **继续创作意图**：提到章节号、小说名称、"继续写"、"下一章"等
   - **单个功能意图**：提到具体功能名称（书名、人设、大纲、诊断等）
   - **新开小说意图**：说"写小说"、"创作故事"或描述创作想法

#### 第二步：根据意图执行

**继续创作意图** → 直接进入场景B，先检查记忆系统：
```
检测到您想继续创作，请提供：

1. 小说名称（如有）
2. 当前进度（写到第几章）
3. 本次目标（继续写下一章/诊断节奏/修改设定/其他）

[系统自动检测 记忆系统/ 的存在]
如果搜索 `*/记忆系统/bible/story_bible.md` 找到多个项目：
→ 列出所有项目名称让用户选择
如果 记忆系统/ 存在（仅 1 个匹配）：
→ 自动读取 bible/ 5文档 + 最新 phases + 最近 5 summaries
→ 组装 L4 上下文包，快速恢复创作状态
如果 记忆系统/ 不存在：
→ **直接终止**，提示用户：「记忆系统未初始化，请先执行功能18初始化。项目创建请走场景A：新开小说。」
→ 内存记忆系统初始化在此处处理，不推迟到 05_26。
```

**单个功能意图** → 直接进入场景C：
```
检测到您需要[功能名称]功能，正在加载...
[直接执行对应功能模块]
```

**新开小说意图或无明确意图** → 显示简化引导：
```
您好！我是小说创作助手，请问：

A. 新开小说 - 从零开始创作
B. 继续创作 - 接着写已有项目
C. 单个功能 - 使用特定功能

请回复 A/B/C，或直接告诉我您的需求。
```

#### 第三步：等待用户响应

在用户做出选择或提供信息前，不得执行任何创作任务。

**禁止行为**：
- ❌ 直接开始创作内容
- ❌ 直接输出小说设定或章节
- ❌ 跳过意图识别直接进入功能
- ❌ 当用户意图明确时仍强制显示完整菜单

## 写前检查点
当执行「继续」/「下一章」时，按以下顺序**自动执行**，失败自愈不询问：

```
步骤A：搜索项目记忆系统
  → 未找到项目 → 转场景A新开项目；找到多个 → 列出让用户选择

步骤B：读取记忆系统（bible×5 + summaries + phases）
  → 缺文档 → 自动初始化补齐，重新读取确认全部存在

步骤C：执行写前预检（加载05_24叙事质量管理框架）
  → 缺字段 → 自动阻断并回溯来源修复 → 重新预检通过后继续

步骤D：加载05_06单章写作模块 → 开始写作（05_06内含章节细纲+角色登记步骤）
```

### 写后流水线

章节保存后，写后流水线以 `modules/05_26_功能模块.md` 为**唯一权威定义**，本协议不重复展开。流程：字数确认→圣经×5+摘要→7个子代理审核（五道门禁/强约束/逻辑因果审查/读者审稿/连贯性/节奏/质量）→修复→.done标记。

## 快捷命令定义

> **⚠️ 权威来源**：以下为流水线上下文提取。完整触发词 → 模块映射以 `SKILL.md` 的「智能触发规则」表为唯一权威。本处仅保留与流水线执行直接相关的命令说明。

将「继续」拆分为两组细粒度的触发词。写后流水线详见 `modules/05_26_功能模块.md`。

### 写作命令
| 触发词 | 执行范围 |
|--------|---------|
| 「继续」/「下一章」 | 上下文恢复 → 写前预检 → 单章写作 → 字数检查 → **自动触发写后流水线（子代理审核闭环）→ 全部 PASS 后继续下一章** |
| 「第X章」 | 同上，定位到第X章续写/重写 |
| 「重写第X章」 | 覆盖指定章节，其余同上 |

### 写后命令
| 触发词 | 执行范围 |
|--------|---------|
| 「写后」 / 「收尾」 | 执行完整写后流水线（详见 `modules/05_26_功能模块.md`）：字数确认→圣经×5+摘要→7子代理审核（五道门禁/强约束/逻辑因果审查/读者审稿/连贯性/节奏/质量）→修复→.done标记 |
| 「半程收尾」 | 仅执行：圣经更新 + 摘要 + 字数确认（快速模式，用于用户催更） |
| 「补缺」 | 检查并补全写后流水线中遗漏的步骤（根据 .done 标记判定） |

### 智能触发规则
- 用户说「继续」写完后 → **自动连续执行**：单章写作 → 字数检查 → 写后流水线（子代理审核闭环）→ 全部 PASS → 立即续写下一章
- 用户说「继续」后紧接着说「下一章」→ 连续写作模式，每章都自动触发完整写后流水线（非每3章）
- **暂停条件**：仅当用户长时间未交互时，在本章全部 PASS 后暂停并等待用户指令。其余情况不停顿、不询问。

## 🔄 每次功能调用必须执行

当用户选择某个功能模块后：
1. **加载模块**：读取对应功能模块文件
2. **读取对应参考资源**：按模块定义加载 `references/` 下的文件
3. **使用对应模板**：按模块定义加载 `templates/` 下的文件
4. **按流程执行**：严格遵循模块的「执行流程」步骤

### 💾 所有产出必须保存

- 设定类内容 → 保存为 `.md` 文件
- 正文类内容 → 保存为 `.txt` 文件
- **禁止仅输出在对话中不保存文件**

### 🔧 错误处理

当遇到错误时：
1. **模块缺失**：提示用户并提供替代方案
2. **路径错误**：尝试自动修复或降级到基础功能
3. **用户输入不明确**：显示简化引导而非报错

### ⚡ 快捷命令一览

| 分类 | 触发词 | 执行范围 |
|------|--------|---------|

| 写后 | 「写后」「收尾」「半程收尾」「补缺」 | 执行/补齐写后流水线各步骤 |

## 🛑 特殊规则：LLM 不按流程执行时的兜底

当 LLM 因自身局限跳过或漏执行某步骤时，以下规则生效：

1. **用户指出漏步** → 立即停止当前动作，执行 `补缺` 流程
2. **.done 标记检测到断裂** → 从断裂点自动回溯补齐，无需用户再次指示
3. **跨会话时.done目录不完整** → 以最新一章为准，补全缺失标记后继续
4. **「继续」后未触发写后流水线** → 自动在字数检查完成后直接触发写后流水线（子代理审核闭环），不询问
