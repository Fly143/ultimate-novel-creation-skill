### 22. 强约束系统

**功能**：叙事参谋，不替作者做决定。在每章写作后自动检查三个维度——金手指叙事健康度、时间流逝感与紧迫度、多条叙事线的读者预期——发现潜在风险时给出提醒，不报「违规」。

**定位**：预防性诊断。不同于功能 17（事后检查一致性），本系统是**事前提醒**——在问题还没形成之前就点出来。与功能 24（叙事质量管理框架）互补——24 侧重写后硬性复核，22 侧重长期趋势追踪。

**触发方式**：
- **自动**：每章写作后，由 `modules/05_26_功能模块.md` 步骤4 子代理B 自动执行
- **手动**：`金手指` / `时间约束` / `叙事线` / `强约束` → 加载本模块并定位到对应子系统

**存储位置**：
- `[书名]/记忆系统/constraints/cheat_constraints.md` — 金手指叙事状态
- `[书名]/记忆系统/constraints/time_constraints.md` — 时间约束追踪
- `[书名]/记忆系统/constraints/narrative_threads.md` — 叙事线监控

## 子系统 1：金手指叙事约束

### 追踪维度

| 维度 | 检查内容 | 风险信号 |
|------|---------|---------|
| **代劳率** | 金手指是否在替主角解决问题？主角自身成长（智慧/毅力/人际关系）推动了什么？ | 近 5 章中金手指解决问题次数 > 主角自身成长次数 × 2 |
| **代价感知** | 最近几章使用金手指时，读者看到代价/消耗/副作用了吗？ | 连续使用 3 次以上无代价描写 |
| **新鲜感** | 展示方式是否重复？最近几次都是同一种套路？ | 连续 3 次使用方式高度相似 |
| **膨胀匹配** | 金手指的解决能力，和当前剧情压力对等吗？ | 金手指轻松解决本应棘手的难题 |
| **来源闭环** | 近期的使用，还跟金手指的来源/设定自洽吗？ | 使用方式超出设定的规则/限制 |

### 初始化

**触发条件**：
- 用户说 `金手指` → 加载本模块
- 首次自动检查时发现 `cheat_constraints.md` 不存在

**初始化流程**：
```
1. 从 story_bible.md 中提取金手指相关信息：
   - 名称、类型、来源
   - 规则与限制
   - 当前阶段/形态
2. 读取 character_arcs.md 中主角的能力演化线
3. 从已有章节中提取最近的金手指使用记录
4. 按模板创建 constraints/cheat_constraints.md
   - 模板位置：templates/constraints/cheat_constraints_template.md
5. 不询问用户，自动完成初始化
```

### 每章写作后检查流程

```
1. 【读取】读取 constraints/cheat_constraints.md
   - 如不存在 → 执行上述初始化流程，创建追踪文档
   - 不询问用户，默认初始化

2. 【分析本章】
   - 金手指在本章中使用了几次？
   - 每次解决了什么问题？
   - 有没有代价/消耗/副作用的描写？
   - 展示方式是什么？
   - 解决的能力等级 vs 当前剧情压力等级
   - 使用方式是否符合已设定的规则？

3. 【计算趋势】
   - 统计最近 5 章中：金手指代劳 vs 主角自身成长 的比例
   - 检查连续使用的代价描写频率
   - 检查使用方式的重复度

4. 【更新记录】
   - 在 cheat_constraints.md「最近 5 次使用」表格中追加本章记录
   - 更新「近期模式摘要」段落
   - 更新「叙事评估」段落的 5 个维度值

5. 【输出】（仅在有风险时输出）
   - 叙事风险提示，不是违规报告
   - 每个风险点附带：具体问题描述 + 建议
```

### 输出示例

```
⚠️ 金手指叙事提示 — 第 27 章后

• 代劳倾向：近 5 章中，主角靠金手指解决问题 7 次，靠自身智慧/毅力仅 2 次
  读者可能感觉「金手指才是主角」
→ 建议：接下来 2-3 章安排一次主角脱离金手指、靠自身能力解决的危机

• 零代价：金手指连续使用 6 次，读者没看到任何代价或消耗
  读者可能觉得「反正随便用，剧情没悬念」
→ 建议：下次使用时让读者感受到冷却/消耗/副作用

• 新鲜度衰减：最近 3 次使用都是「默念口诀→金光一闪→解决」
  套路化会让爽感打折
→ 建议：换个展现方式，或者让使用过程出一次意外
```

### 零风险静默规则

如果本章金手指使用在代劳率、代价感知、新鲜感、膨胀匹配、来源闭环 5 个维度上均无风险信号，**不输出任何内容**。子系统静默通过，不刷存在感。

## 子系统 2：时间约束

### 追踪维度

| 维度 | 检查内容 | 风险信号 |
|------|---------|---------|
| **时间流逝感** | 最近几章过去了多久？读者感受和故事内时间一致吗？ | 章节数增长快但故事内时间几乎不动；或大量时间跳跃无叙事交代 |
| **紧迫度叙事** | 如果有 deadline，读者在最近的章节里还能感受到压力吗？ | deadline 已设定但连续 5+ 章未提及压力；压力线断裂 |
| **节奏匹配** | 大事前夕是否写了太多日常？还是赶太急跳过了该有的铺垫？ | deadline 临近却插入大量无关日常；或无铺垫直接跳到高潮 |

### 初始化

**触发条件**：
- 用户说 `时间约束` / `deadline` → 加载本模块
- 首次自动检查时发现 `time_constraints.md` 不存在

**初始化流程**：
```
1. 从 story_bible.md 中提取时间相关信息：
   - 故事起始时间
   - 当前进度
2. 从 plot_timeline.md 中提取已有的时间线事件
3. 从已有章节中提取时间流逝记录
4. 检查是否有用户设定的 deadline（如大纲中提到的截止事件）
5. 按模板创建 constraints/time_constraints.md
   - 模板位置：templates/constraints/time_constraints_template.md
6. 不询问用户，自动完成初始化
```

**添加 deadline**：
- 大纲阶段或用户随时说「加个 deadline：30 天后宗门大比」
- 自动追加到 time_constraints.md「活跃的时间约束」表格

### 每章写作后检查流程

```
1. 【读取】读取 constraints/time_constraints.md
   - 如不存在 → 执行上述初始化流程，创建空追踪文档
   - 不询问用户

2. 【更新流逝】
   - 分析本章故事内过了多久（天/小时/月）
   - 计算累计总时间
   - 计算距离各 deadline 的剩余时间
   - 在时间线记录表格中追加本章记录

3. 【检查紧迫度】
   - 是否有 deadline 正在逼近？
   - 如果有：最近的章节是否在叙事上维持了这份紧迫感？
   - deadline 距离超过 10 章未被提及 → 标记为风险

4. 【检查流逝比例】
   - 故事内时间流速和章节进展比例是否合理？
   - 200 章只过了 3 天 → 读者可能感觉时间停滞
   - 3 章过了 10 年 → 读者可能感觉跳跃太急
   - 对比同类型小说的常见节奏

5. 【更新评估】
   - 更新 time_constraints.md「叙事评估」段落

6. 【输出】（仅在有风险时输出）
```

### 输出示例

```
⚠️ 时间叙事提示 — 第 35 章后

• 宗门大比（第 20 章埋下，说「三个月后」）
  已过 15 章，故事内仅过 5 天
  读者的「deadline 思维」已经模糊
→ 建议：后续章节加快时间流速，或加入时间跳跃

• 紧迫度断裂：第 30-32 章还在渲染大比压力，第 33-35 章转入日常线
  压力线断了 3 章，读者可能已经放松
→ 建议：下一章用一个细节（如收到宗门催函）拉回紧迫感

• 节奏倒挂：主角「三天后突破」的紧迫节点，插了一整章咖啡厅闲聊
  读者的焦急预期被稀释
→ 回顾是否可以把闲聊压缩到半章
```

### 零风险静默规则

如果本章时间流逝合理、紧迫度维持良好、节奏匹配正常，**不输出任何内容**。子系统静默通过。

## 子系统 3：多条叙事线监控

### 追踪维度

| 维度 | 检查内容 | 风险信号 |
|------|---------|---------|
| **被遗忘的线** | 某条线超过 N 章没提，且它埋的悬念读者还在等 | 活跃线超过 10 章未涉及；悬念高潮处停留超过 15 章 |
| **情绪单调** | 所有活跃线最近都是同一情绪基调？读者需要喘息 | 近 5 章所有活跃线全是压抑/紧张；或全是日常/轻松 |
| **预期负债** | 某条线卡在悬念处太久，读者的等待在贬值 | 伏笔悬而未决超过 15 章；承诺的后续无兑现迹象 |

### 初始化

**触发条件**：
- 用户说 `叙事线` / `支线监控` → 加载本模块
- 首次自动检查时发现 `narrative_threads.md` 不存在

**初始化流程**：
```
1. 从 plot_timeline.md 中提取所有叙事线（主线 + 支线）
2. 从 character_arcs.md 中确认各线的 POV/主角
3. 从 chekhovs_gun.md 中提取各线相关的伏笔
4. 从已有章节中确定各线的最后活跃章节和情绪基调
5. 按模板创建 constraints/narrative_threads.md
   - 模板位置：templates/constraints/narrative_threads_template.md
6. 不询问用户，自动完成初始化
```

### 每章写作后检查流程

```
1. 【读取】读取 constraints/narrative_threads.md
   - 如不存在 → 从已有设定（大纲、人物关系图谱、plot_timeline.md）自动提取叙事线信息
   - 不询问用户

2. 【更新状态】
   - 分析本章涉及了哪些叙事线
   - 更新各线的「最后活跃章节」
   - 在情绪日志中追加本章各活跃线的情绪基调

3. 【检查遗忘】
   - 遍历所有状态为「活跃」的叙事线
   - 某条线超过 10 章没碰 → 提醒「可能被遗忘」
   - 某条线停留在悬念高潮处超过 15 章 → 提醒「读者预期正在贬值」

4. 【检查情绪单调】
   - 统计近 5 章所有活跃线的情绪基调
   - 如果全部相同 → 提醒「情绪节奏单一，读者需要喘息」
   - 特别关注连续 5 章以上同一基调（压抑/紧张尤甚）

5. 【检查预期负债】
   - 对照 chekhovs_gun.md 中与各叙事线相关的伏笔
   - 伏笔悬而未决超过 15 章 → 提醒「预期负债累积」
   - 承诺了但未兑现的后续事件 → 标记为负债

6. 【更新评估】
   - 更新 narrative_threads.md「叙事评估」段落

7. 【输出】（仅在有风险时输出）
```

### 输出示例

```
⚠️ 叙事线提示 — 第 42 章后

• 支线「苏晴的身世」：第 25 章揭了半个悬念后 17 章没碰
  读者还在等，但耐心在递减
→ 建议：3 章内切回去一次，哪怕只给一个细节/伏笔，保持悬念热度

• 情绪单调：主线 + 支线 1「宗门斗争」近 5 章全在压抑/紧张基调
  读者连续紧张太久会疲劳
→ 建议：接下来 1-2 章让某条线出现小爽点或温情戏，给读者呼吸空间

• 线索「林萧的玉佩」：第 15 章后未被任何线提及
  该线索属于主线设定，长期冷落可能导致后期回收时生硬
→ 建议：近期让某个角色不经意提及玉佩，做一次轻量存在感刷新
```

### 零风险静默规则

如果所有活跃叙事线均未被遗忘、情绪基调有变化、预期负债未累积，**不输出任何内容**。子系统静默通过。

## 约束系统与记忆系统的关系

| | 记忆系统（bible/） | 约束系统（constraints/） |
|---|---|---|
| **用途** | 记录事实（发生了什么） | 分析风险（可能会发生什么） |
| **更新方式** | 每章追加 | 每章分析 |
| **性质** | 工具书 | 参谋 |
| **输出** | 不主动说话 | 有风险才提醒 |
| **存储** | `bible/` 目录 5 个文档 | `constraints/` 目录 3 个文档 |

约束系统**不替代**记忆系统，而是在记忆系统之上做一层叙事健康分析。记忆系统提供事实数据，约束系统基于事实做趋势判断。

**流水线模式行为**：
- 子代理 B 始终返回 `PASS`，不返回 `FAIL`，不触发修复
- 无风险 → `PASS`
- 有风险 → `PASS: <风险提示摘要>，已写入约束文件`
- 判断依据：context 中包含 `mode: pipeline` 标识

**手动触发模式**（独立于流水线）：
- 用户说 `强约束` / `金手指` / `时间约束` / `叙事线` → 直接加载本模块
- 手动模式下输出完整风险提示（非 PASS/FAIL 二值），详见各子系统输出示例

## 手动触发命令

| 命令 | 执行 |
|------|------|
| `强约束` / `约束检查` | 执行全部三个子系统的检查（delegate_task） |
| `金手指` / `金手指约束` | 仅检查/管理金手指叙事约束 |
| `时间约束` / `deadline` | 仅检查/管理时间约束 |
| `叙事线` / `支线监控` | 仅检查/管理叙事线监控 |

手动触发时，如果对应约束文档不存在，先执行初始化（从已有设定文档中提取信息），再执行检查。
