# 功能模块 22：版本控制

**功能**：记录创作历史，支持回滚和对比修改。

---

## 版本记录

### 自动保存时机

- 每次修改设定文件后
- 每章创作完成后
- 执行修订优化后

### 版本存储结构

```
[书名]/
├── 设定/
│   └── .versions/           ← 版本目录
│       ├── 人设卡_v1.md
│       ├── 人设卡_v2.md
│       ├── 大纲_v1.md
│       └── version_log.md   ← 版本日志（自然语言）
└── 正文/
    └── .versions/
        ├── 第001章_v1.txt
        └── 第001章_v2.txt
```

---

## 版本日志格式

```markdown
# 版本日志

## 人设卡
- **v1** — 2026-04-11 10:30 — 初始版本（创建主角人设）
- **v2** — 2026-04-11 15:45 — 调整性格设定（主角性格从冷酷改为外冷内热）
```

---

## 快捷命令

- "查看历史" → 显示文件修改历史
- "对比版本" → 对比两个版本的差异
- "回滚到v1" → 恢复到指定版本
- "保存版本" → 手动创建版本快照

---

## 版本对比输出

```
=== 版本对比报告 ===

【文件】：人设卡.md
【对比】：v1 → v2

【变更内容】
- 性格：冷酷 → 外冷内热
- 背景：孤儿 → 家族弃子
- 新增：隐藏能力设定

【建议】
- 性格变更可能影响后续章节人物一致性
- 建议检查已写章节是否需要调整
```

---

## 注意事项

- 版本目录默认隐藏（以.开头）
- 建议重要节点手动保存版本
- 回滚前会自动创建当前版本备份
