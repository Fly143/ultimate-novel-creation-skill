### 12. 小说节奏诊断与优化

**功能**：分析章节内部节奏比例、与前章节奏对比、爽点分布合理性，输出诊断报告并提供优化建议。
**触发方式**：
- 手动：`诊断` / `节奏` / `节奏诊断`
- 写后流水线：子代理E 并行启动

**参考资源**：`references/爽文创作理论.md`（节奏理论基础）
**模板**：`templates/节奏诊断报告.md`

---

## 子代理调用规格

> 本模块必须由 `delegate_task` 启动的独立子代理执行。
> 子代理不共享主代理的写作上下文，避免"自己审自己"的偏见。
> 子代理需独立读取：章节正文、最近5章摘要、章节规划表。

### 调用方式

```
delegate_task(
  goal: "诊断指定章节的节奏问题，输出节奏诊断报告",

  toolsets: ["file"]

```

### 需要读取的文件

| 文件 | 用途 |
|------|------|
| 章节正文 | 诊断对象，分析内部节奏 |
| 最近5章摘要 | 与前章节奏对比，检测单调/跳跃 |
| 章节规划表 | 对照规划检查节奏是否偏离预期 |
| 情节大纲 | 辅助判断节奏是否服务于主线（如需要） |

### 输出要求

- 将诊断报告写入 `{书名}/设定/[书名]_节奏诊断报告_第XX章.md`
- 如为全书诊断则保存为 `{书名}/设定/[书名]_节奏诊断报告_全书.md`
- 报告格式参照 `templates/节奏诊断报告.md`

## 诊断流程

### 步骤 1 — 读取章节与前文摘要

1. 读取当前章节正文全文
2. 读取最近 5 章的摘要（`{书名}/记忆系统/summaries/ch_NNN_summary.md`）
   - 如摘要不足 5 章，有多少读多少
3. 读取章节规划表，确认本章在整体规划中的定位（过渡章/高潮章/铺垫章）

---

### 步骤 2 — 分析章节内部节奏

将本章正文按叙事结构分为 4 个阶段，评估各阶段比例：

| 阶段 | 定义 | 理想比例 |
|------|------|----------|
| **开篇** | 开场钩子、场景建立、情绪基调 | 10-15% |
| **铺垫/上升** | 矛盾引入、压力递增、悬念积累 | 30-40% |
| **高潮** | 核心冲突爆发、情绪峰值、爽点释放 | 25-35% |
| **收尾** | 余波处理、情绪回落、下章钩子 | 15-25% |

**检查要点**：
- 开篇是否有钩子（动作/悬念/冲突）？还是平淡开局？
- 铺垫是否充分？读者能否感受到压力递增？
- 高潮是否与前面的铺垫匹配？铺垫500字→高潮1000字 = 头重脚轻
- 收尾是否干脆？有无拖沓？是否留下下一章的悬念？
- 各阶段之间是否有自然过渡？

**节奏问题判定**：

| 问题类型 | 判定标准 |
|----------|----------|
| 🟡 节奏过快 | 开篇<10% 且铺垫<25%，读者来不及进入状态就到了高潮 |
| 🟡 节奏过慢 | 铺垫>50%，高潮<15%，大量文字没有推进剧情 |
| 🟡 头重脚轻 | 高潮>45% 且铺垫<20%，缺乏积累就直接爆发 |
| 🟡 尾巴过长 | 收尾>30%，高潮后大量余波拖沓 |
| 🟢 节奏均衡 | 各阶段比例在理想范围内 |

---

### 步骤 3 — 与前章节奏对比

对比本章与最近 5 章的节奏模式：

**检测单调**：
- 连续 3 章以上节奏模式相同（如都是"慢→慢→慢"或"快→快→快"）
- 连续章节高潮强度无变化（情感疲劳）
- 每章结尾都是同一类型的钩子（读者产生预期后失去兴趣）

**检测跳跃**：
- 前 3 章铺垫节奏→本章突然高潮（读者没有准备）
- 前 3 章高潮密集→本章突然平缓（情绪断崖）
- 场景转换过快，上一章结尾和本章开头之间缺乏过渡

**节奏波动评估**：
```
理想节奏波形（示例）：

  高潮  ╱╲      ╱╲╲
       ╱  ╲    ╱    ╲
  铺垫╱    ╲  ╱      ╲
     ╱      ╲╱        ╲
  低谷                    ╲╱

  → 张弛有度，波峰波谷交替出现
  → 相邻章节之间有节奏差异
  → 整体趋势向上（后期高潮强度递增）
```

**对比结论**：
- 🟢 节奏多样：与前章有明显差异，张弛有度
- 🟡 节奏单调：与前 2+ 章节奏模式雷同
- 🟡 节奏跳跃：与前章节奏反差过大，缺乏过渡

---

### 步骤 4 — 检查爽点分布

1. 识别本章所有爽点（参照 `references/爽文创作理论.md` 的爽点类型）
2. 标记每个爽点在章节中的位置（前/中/后）
3. 评估分布合理性：

| 检查项 | 标准 |
|--------|------|
| 爽点数量 | ≥1 个爽点（过渡章可为 0，但连续 2 章为 0 需警告） |
| 爽点位置 | 不应全部集中在开头或结尾，中段必须有 |
| 爽点类型 | 应与本章定位匹配（铺垫章→小爽点；高潮章→大爽点） |
| 爽点间隔 | 连续爽点间隔不宜超过 1500 字（网文读者耐心有限） |
| 爽点与节奏匹配 | 高潮阶段应有爽点释放，铺垫阶段应积累期待感 |

**爽点密度评分**：
- 🟢 健康：1-3 个爽点，分布均匀，类型匹配
- 🟡 偏低：0 个爽点（非过渡章）或全部集中在末尾
- 🟡 过密：4+ 个爽点，读者审美疲劳
- 🔴 错位：铺垫章放了大爽点，或高潮章无爽点释放

## 常见节奏问题速查表

| 现象 | 可能原因 | 解决方向 |
|------|----------|----------|
| 读者反馈"看不下去" | 开篇无钩子，铺垫过长 | 前300字必须有冲突/悬念/动作 |
| 读者反馈"太赶了" | 铺垫不足，高潮缺乏积累 | 增加1-2个压力递增场景 |
| 读者反馈"这章好水" | 无效描写过多，无实质推进 | 删除重复描写，增加剧情推进 |
| 读者反馈"看不懂" | 场景转换太快，缺过渡 | 增加衔接段落，交代清楚空间/时间转换 |
| 读者反馈"太平了" | 无高潮，全章铺垫 | 在中后段安排一个小高潮或反转 |
| 读者反馈"又是一样的" | 连续章节节奏模式雷同 | 变换节奏组合（快慢交替） |
| 读者反馈"戛然而止" | 收尾太急，余波未处理 | 增加收尾段落，但不超过25% |
| 读者反馈"后面好拖" | 收尾过长，重复已知信息 | 精简收尾，留下钩子即可 |

## 注意事项

1. **诊断时机**：建议每 5 章执行一次节奏诊断，或在用户反馈节奏问题时立即执行
2. **过渡章宽容**：过渡章/铺垫章允许低爽点密度，但不可连续 2 章无爽点
3. **高潮章要求**：高潮章必须有≥1 个爽点释放，且收尾不可超过 25%
4. **模板灵活性**：如果本章是特殊结构（如闪回章、多视角章），可在报告中额外说明，不强行套用标准模板
5. **结合读者审稿**：节奏诊断结果应与读者审稿（05_25）的"吸引力与可读性"评分交叉验证
