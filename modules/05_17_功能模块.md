### 17. 连贯性检查

**功能**：全局一致性深度审查。对照 v5.0 记忆系统圣经文档，逐项检查 100 章以上长篇的人物/剧情/伏笔/时间线连贯性。

**参考资源**：`references/连贯性维护指南.md`
**记忆系统**：直接对照 `[书名]/记忆系统/bible/` 4 个圣经文档执行检查

---

**执行流程**：

```
=== 连贯性检查阶段（v5.0 自然语言记忆） ===

1. 【检查记忆系统】
   - 确认 [书名]/记忆系统/ 目录是否存在
   - 如果不存在：提示用户「记忆系统未初始化，建议先使用功能18初始化记忆系统后再执行连贯性检查」，终止本模块
   - 如果存在，继续执行：

2. 【加载记忆上下文】
   - 读取 [书名]/记忆系统/bible/story_bible.md → 了解世界观和当前状态
   - 读取 [书名]/记忆系统/bible/character_arcs.md → 了解所有角色演化线
   - 读取 [书名]/记忆系统/bible/plot_timeline.md → 了解事件因果链和时间线
   - 读取 [书名]/记忆系统/bible/chekhovs_gun.md → 了解伏笔状态

3. 【人物一致性检查】（对照 character_arcs.md）
   - 性格：最近 5 章角色行为是否符合演化线
   - 能力：实力增长是否符合设定速度
   - 关系：角色关系变化是否有铺垫
   - 对话：口头禅/说话风格是否一致
   - OOC 风险标记：偏离弧线的行为 → 记录偏差

4. 【剧情连贯性检查】（对照 plot_timeline.md）
   - 事件因果：最近事件是否与之前事件逻辑矛盾
   - 时间线：时间跨度是否合理（无倒流、无跳跃无解释）
   - 支线状态：各支线是否被遗忘
   - 爽点分布：对比章节规划表，检查爽点密度

5. 【伏笔完整性检查】（对照 chekhovs_gun.md）
   - 遍历所有未回收伏笔，检查是否被遗漏
   - 回收窗口：接近回收期限的伏笔 → 标记为「即将过期」
   - 矛盾检查：新内容是否与已埋伏笔冲突

6. 【世界观一致性检查】（对照 story_bible.md）
   - 规则冲突：新章节是否违反已设定的世界规则
   - 势力变动：势力格局变化是否有合理解释
   - 物品/能力一致性：关键道具和能力是否出现矛盾

7. 【生成连贯性报告】
   - 保存至 [书名]/设定/连贯性检查报告.md
   - 输出问题等级：🟢 正常 / 🟡 注意 / 🔴 需修复
   - 每个问题附带：具体章节号 + 圣经文档对应条目 + 修复建议
```

**检查深度**：
- **基础检查**（每次调用）：遍历 4 个圣经文档，标记明显矛盾
- **深度检查**（每 20 章建议执行）：逐章回溯 + 全文交叉验证
- **紧急检查**（用户说"跑偏了"）：聚焦问题章节 vs 圣经文档

**输出**：
- 连贯性检查报告：`[书名]/设定/连贯性检查报告.md`

